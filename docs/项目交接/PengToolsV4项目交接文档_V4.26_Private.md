## 1. 项目定位与交接范围

PengTools 是一个基于 Windows 的离线桌面工具集合，服务于日常开发、运维、需求归档、SQL 发版准备、接口文档维护与个人知识管理。当前交接版本为 **V4.26 Private（私人版）**，在标准工具能力之上增加了：

- 需求管理、日报、升级准备之间的联动；
- 本地需求/SVN 工作副本归档、目录扫描和文件浏览；
- 多系统 SQL 与生产发版 Excel 自动生成；
- 自我学习知识库与 Excel/Word/TXT 等文件的内置查看编辑；
- 自我学习模块的彩蛋解锁（日报、需求管理不隐藏）；
- 所有用户数据落盘到安装目录 `data`，使程序升级不影响历史资料。

本交接文档描述的是**当前已经实现的代码边界与维护规则**。它不保证可连接用户单位内网 SVN；当前开发环境未对内网地址做真实性验证，SVN 能力仅做过本地临时仓库逻辑验证。

## 2. 技术架构

### 2.1 技术栈

| 层级 | 技术/组件 | 说明 |
|---|---|---|
| GUI | Python + PyQt6 | 纯桌面端，窗口、树、表格、对话框、线程与托盘均由 PyQt6 实现 |
| Excel | openpyxl | 生产发版清单生成、学习模块 Excel 读取/导出 |
| Word | python-docx | 接口 DOCX 更新、自我学习 Word 读取/编辑/导出 |
| 加密 | gmssl | 网关 SM2 + SM4 国密解密 |
| 加密 Excel | msoffcrypto-tool | 支持带密码的 Excel 导入（使用者提供密码） |
| 打包 | PyInstaller | `--onefile --windowed` 单 EXE Windows 包 |
| 样式 | QSS | 全局样式位于 `resources/style.qss` |
| SVN | 本机 SVN 命令行 | 经 `subprocess` 调用；需要安装 TortoiseSVN 的 command line client tools |

安装依赖见 `requirements.txt`。开发时建议使用独立 Python 虚拟环境；发布构建机器还需要 `pyinstaller`。

## 3. 本地数据、升级兼容与安全边界

### 3.1 数据目录规则

`config.local_data_dir()` 是唯一的数据目录判定入口：

- 开发运行：`PengToolsV4/data/`；
- PyInstaller 打包运行：`<PengToolsHub_Private.exe 所在目录>/data/`；
- 不依赖用户 Windows 用户目录，不应把工作数据写到 `_MEIPASS` 临时目录。

这条规则使用户可以直接覆盖 EXE 完成升级，而保留全部资料、日报、需求、系统配置和 UI 布局。

### 3.2 数据文件字典

| 文件/目录 | 所属模块 | 内容 | 兼容要求 |
|---|---|---|---|
| `systems.json` | 系统配置、需求、SQL、升级准备 | 系统名称、SQL 标题、SVN 系统目录、账号/环境名、路径模板 | 新增字段使用 `setdefault` 兼容旧数据；删除系统前提示需求可能需重选 |
| `settings.json` | 设置/主窗口 | 字号、语言、悬浮栏、关闭行为、保活参数 | 通过 `normalize_settings()` 限制范围并补默认值 |
| `ops_custom_commands.json` | 运维助手 | 用户自定义运维命令 | 保持 JSON 列表/对象结构的向后兼容 |
| `private_knowledge.json` | 自我学习 | 分类、文本、Excel 行列数据、Word 内容及来源 | 不能仅存展示文本；Excel 保留结构化行列数据 |
| `daily_reports.json` | 日报 | 每日已完成、明日计划、风险备注等 | 日期键应使用 `yyyy-MM-dd` |
| `daily_report_settings.json` | 日报 | 提醒开关、提醒时刻、当天已提醒标识等 | 默认每天最多提醒一次 |
| `requirements.json` | 需求管理、升级准备、日报 | 需求/BUG 台账、路径、标记、SQL、日期 | 旧记录缺字段时按默认值读取，不能因字段缺失崩溃 |
| `requirement_ui.json` | 需求管理 | 左右、上下 splitter 尺寸 | 仅保存两个正整数，默认 `[355,700]` 和 `[360,210]` |
| `svn_workspaces/` | SVN 工作副本 | 可由需求管理创建/管理的工作副本 | 视为用户数据，升级不可删除 |

### 3.3 升级与发布的硬规则

1. 任何私人功能改动只能重新构建私人包，**不得改写标准包**。
2. 安装包/更新包不得包含既有用户 `data`。构建脚本会清理 `PrivateInstaller/data`，这是必要保护，不要移除。
3. 升级交付方式是替换 EXE、`README.txt` 等程序文件；不得删除、复制或覆盖安装目录的 `data`。
4. 新增数据字段必须“读旧兼容”：读取时填默认值，保存时保留已知旧字段；不可假定所有历史 JSON 都有新字段。
5. 删除数据必须二次确认：取消按钮在左、确认删除在右，默认焦点在取消，避免回车误删。

## 4. 模块功能说明

### 4.1 工作台

作为入口和常用功能入口页。后续扩展时保持“低操作成本”原则：可跳转到具体业务模块，不要在工作台复制一套业务编辑逻辑。

### 4.2 证件类型

用于个人/单位证件测试数据模拟。属于独立工具模块，不应与需求、SQL 或用户知识库共享敏感测试数据。

### 4.3 升级准备（原 SQL 整理）

文件：`panels/sql_panel.py`、`tools/release_prep.py`。

这是当前最重要的交付模块，包含三个业务页：

1. **升级准备**：选升级日期，推荐需求/BUG，勾选后生成 SQL 包和生产发版清单；
2. **SQL 整理**：粘贴/导入 SQL，识别 DDL/DML，组织升级 SQL、回滚 SQL、验证 SQL，按目录模板导出；
3. **系统配置**：维护多个系统的 SQL 标题、提交目录、账号、环境名称与路径模板。

关键行为：

- 必须先确认升级日期再生成，避免误用过期推荐列表；
- 候选需求按“实际/计划上线日恰好匹配 > 同月匹配 > 其他日期 > 无日期”排序；
- 每条需求可选所属系统；多系统场景按系统分别生成 SQL；
- 开发分支 SVN 地址末段会自动提取为发版 Excel 的“分支名称”；缺失开发分支时允许保存与生成，Excel 分支列留空；
- 额外粘贴 SQL 在选中多系统时必须显式选择归属系统，避免脚本投递错误；
- 验证 SQL 不进入 SVN 提交目录，作为发版前后留存与验证材料；
- 生产发版 SVN 固定默认地址定义在 `tools/release_prep.py`：`svn://10.128.23.145:13690/YDPIC_SharingCenter/17.集成测试/02-生产环境发版任务清单`。

#### 发版 Excel 写入规则

资源模板：`resources/release_workbook_template.xlsx`。逻辑：

- 升级日转换为 `yyyyMMdd`，例如 `2026-07-23` -> `20260723`；
- 若模板有同名 Sheet，定位该 Sheet 最后一条有效数据后追加；
- 若没有同名 Sheet，从最近的日期 Sheet 复制得到新 Sheet，复制列宽、行样式、表头和布局，清空数据区；
- 写入 23 个固定列，计划升级日期保留 Excel 日期格式 `yyyy-mm-dd`；
- 标题行必须和 `RELEASE_HEADERS` 完全一致，模板变更导致不一致时应明确报错，不能静默错列。

### 4.4 接口 DOCX 更新

文件：`panels/docx_panel.py`。用于根据 SQL 结构变化辅助更新接口 Word 文档。后续修改应保持“SQL 结构变化 -> 文档字段/描述更新”的关联，不要把它变成不可追踪的纯文本替换工具。

### 4.5 车辆 VIN

文件：`panels/vin_panel.py`。提供中国车辆 VIN 测试数据能力，作为独立数据工具维护。

### 4.6 网关解密

文件：`panels/gateway_panel.py`。处理 SM2 + SM4 国密报文解密。密钥、明文和报文均可能敏感；后续不要默认持久化到 `data` 或日志。

### 4.7 运维助手

文件：`panels/ops_panel.py`。提供 Linux 运维命令查询、安全引导和自定义命令。自定义内容存入 `ops_custom_commands.json`，命令展示应同时说明风险，避免软件直接自动执行破坏性命令。

### 4.8 设置、托盘、悬浮工具栏与关闭行为

文件：`panels/settings_panel.py`、`main_window.py`、`config.py`。

设置位于左侧导航底部，与模块标题不混排。当前可配置：字号、语言、悬浮栏透明度/置顶/启动显示、复制反馈时间、关闭询问、默认关闭动作、保活开关和间隔。

关闭流程是完整用户功能而非单一 `closeEvent`：

- `close_ask_each_time=True` 时询问最小化或退出；
- 默认动作为最小化；
- 托盘“退出”必须走强制退出路径，不能再次最小化；
- `QApplication.setQuitOnLastWindowClosed(False)` 允许窗口隐藏后继续提供托盘与提醒服务。

### 4.9 自我学习（私人隐藏模块）

文件：`panels/personal_panel.py` 的 `KnowledgeTab`。

定位是“懒人化的本地学习资料库”，支持粘贴和文件导入，按内容自动分类、全文检索、结构化查看及再次编辑。支持格式：Excel、Word、TXT、Markdown、SQL、JSON。

重点能力：

- 直接粘贴文本后自动抽取分类，允许人工修改分类、标题和内容；
- 搜索框实时给出提示；筛选/定位结果以高亮方式展示；
- Excel 不允许退化为整段文本：应按单元格/行列结构展示；
- Excel 可实时定位和高亮命中位置、隐藏选中行/当前列、恢复隐藏项、双击单元格复制、编辑单元格、复制整行、复制当前展示、复制整表、导出当前展示、导出整表；
- Word 可在内置编辑器中保存和导出；
- 内置资料也允许修改更新；带密码 Excel 在导入时由用户输入密码；
- 删除同样遵循“取消默认”的二次确认规则。

#### 私人解锁规则

当前只有“自我学习”隐藏。彩蛋解锁后显示该导航项。日报和需求管理已明确改为常显，后续不得一起藏回去。

密钥规则沿用当前实现，维护入口为 `MainWindow._unlock_private_tools()`。因该机制属于私人授权约定，后续开发只能在用户明确要求时修改，不要擅自改密钥或把解锁方式泄露到普通 UI 文案中。

### 4.10 日报

文件：`panels/personal_panel.py` 的 `DailyReportTab`。

能力包括：日报日期选择、历史日报读取、已完成/明日计划/备注编辑、保存、复制、删除、提醒设置。提醒通过应用托盘通知，同一天只提醒一次。

与需求管理联动：从需求详情选择“加入今日日报”后，调用 `tools.requirements.daily_template()` 自动生成可编辑的日报片段。模板会携带需求/BUG 编号、标题、分类、状态以及 SQL/接口/周边通知/临时升级标记。它是草稿生成器，不应覆盖用户已经手工写入的日报内容。

### 4.11 需求管理

文件：`panels/requirement_panel.py`、`tools/requirements.py`、`tools/svn_workspace.py`。

需求管理是需求、BUG、资料目录、SVN、SQL、日报、升级准备的中枢。记录主要字段为：编号、可编辑标题、类型、描述、分类、状态、优先级、系统、负责人、计划/实际/月份日期、SQL/接口/周边/临时标记、SQL 片段、来源文件、SVN 路径、本地目录、版本和扫描信息。

#### 左侧树

- 顶层按上线月份/目录月份分组，月份完整显示、深色较大字体、可折叠；
- 子项展示可编辑的目录标题（通常由 SVN 或本地路径末段得到），需求号/BUG 号做高亮；标题与标记颜色应较浅，避免所有字都加粗造成层级混乱；
- 顶部提供全选复选框、展开、折叠；支持多选删除；
- 子项可拖动至其他月份分类。拖动使用延迟更新，避免节点消失；若原计划上线日期为空，拖到月份时补该月最后一天；
- 支持右键修改标题、打开文件夹；禁止让右键菜单事件误触发树拖动；
- 列表按月份倒序；同月通常按源文件最新修改时间倒序。

#### 右侧详情与文件树

- 保留 SVN 路径、本地文件路径、完整文件树、关联 SQL；不再保留无用途的“需求附件”说明框；
- 文件树展示 **名称、修改时间、类型、大小**，文件夹自动展开，列边界可见、列宽可拖动、列可移动；
- 文件/目录支持双击、右键打开，右键“打开所在文件夹”可用于定位；
- 不再展示不可靠的 SVN 状态列；只有应用成功锁定的文件在名称前显示锁图标；
- 左右区域与右侧“文件树/关联 SQL”上下区域均为 splitter，用户拖动尺寸写入 `requirement_ui.json`，不能被加载状态重新挤动。

#### 录入、导入与搜索

- 可新增需求、编辑需求、粘贴 BUG；开发分支 SVN 地址允许留空；
- 新增/编辑可绑定已有 SVN 工作副本或普通本地资料目录；选择 SVN 工作副本时尝试自动回填 SVN 地址；
- 支持扫描需求根目录，识别普通目录和 SVN 工作副本，并识别“2026年6月”“2026-06”等月份目录；Office 临时文件会忽略；
- 支持按文本搜索，搜索范围含编号、标题、说明、系统、状态、路径、SQL、来源文件及 Excel 行列数据；
- “SQL、需通知周边系统升级、接口文档、临时升级”是关键标记，需在左侧子项中可见，并成为升级/日报处理依据；
- 上线月份、计划上线、实际上线均支持手动录入和日期选择弹窗，日期 UI 与全局 QDateEdit 样式一致。

#### SVN 操作与锁定

支持检出、扫描、更新、拉取、提交、锁定、解锁等。实现通过本机 `svn` 命令行：

- 未安装命令行 SVN 时会给出安装 TortoiseSVN command line tools 的明确提示；
- SVN URL 必须以 `http`、`https`、`svn`、`svn+ssh` 或 `file` 开头；
- 锁定/解锁仅对文件有效；锁图标仅反映软件成功执行过的锁定，不应伪造全仓库状态；
- 内网 SVN 地址无法在当前环境验证，不能在交付或 UI 中声称已经联通；
- 本地临时 SVN 仓库已经验证过 checkout、扫描、更新、锁冲突、解锁和提交的基础逻辑。

## 5. 核心联动流程

### 5.1 需求 -> SQL -> 升级准备 -> 发版 Excel

```text
导入/扫描/新增需求或 BUG
        ↓
维护系统、上线日期、开发分支 SVN、SQL 与升级标记
        ↓
升级准备选择升级日期，自动推荐需求/BUG
        ↓
勾选本次升级项，逐项确认所属系统和发版内容
        ↓
按系统生成 SQL（升级/回滚/验证） + 按模板写生产发版 Excel
        ↓
必要时在 SVN 工作副本提交 SQL；验证 SQL 留在交付目录外
```

需要特别注意：需求可没有开发分支。此时仍允许保存和生成发版材料，只是分支名称为空；不可因为该字段为空阻塞整个升级准备。

### 5.2 需求 -> 日报

```text
在需求详情选择加入今日日报
        ↓
生成带编号、标题、状态与升级标记的日报草稿
        ↓
用户继续编辑“今日完成 / 明日计划 / 备注”
        ↓
保存到 daily_reports.json，按提醒时间由托盘提示填写
```

### 5.3 资料 -> 自我学习

```text
直接粘贴 或 导入 Excel/Word/TXT/Markdown/SQL/JSON
        ↓
提取正文和结构化数据，自动建议分类
        ↓
用户可编辑、实时搜索定位、高亮查看、复制/导出
        ↓
保存到 private_knowledge.json，原文件可被更新
```

## 6. UI、Loading 与交互约束

### 6.1 全局样式

全局 QSS 位于 `resources/style.qss`。目前 QComboBox 和 QDateEdit 明确增加了下拉箭头区域、蓝色 SVG 箭头、悬停/焦点状态，避免用户看不出它们是可下拉控件。后续新增下拉框和日期控件要复用这一套对象样式，不要另起不一致的控件皮肤。

设置入口在左侧底部；右侧内容区不应重复一个无用标题。需求管理列边框、文件树表头和 splitter 要保持可见且可辨识，视觉目的在于减少误操作而不是增加装饰。

### 6.2 Loading 规则

Loading 是不占布局的浮层，不应把按钮、右键菜单、树控件顶开或导致 splitter 尺寸变化。`RequirementPanel._start_task(..., show_loading=False)` 已支持后台静默任务。

**应显示 Loading 的耗时动作：**

- 扫描需求文件夹；
- 导入较大文件或资料；
- SVN 检出、更新、拉取、提交、锁定/解锁；
- 发版材料生成、SQL 导出、实际文件写入。

**不应显示 Loading/不应禁用页面的动作：**

- 文件树后台刷新；
- 普通搜索、筛选、选择树节点、展开折叠、拖动分隔条；
- 右键菜单显示、复制文本、切换单条记录。

扫描完成后 UI 必须恢复可操作；若后续新增异步任务，务必在成功、失败、异常三条路径统一停止 Loading 并恢复状态。

### 6.3 删除与高风险操作

- 所有删除操作（需求、日报、知识、系统配置、文件引用）必须调用统一确认逻辑；
- 对删除、提交、覆盖、解锁等高风险操作，确认文案需明确目标对象和影响；
- 删除确认框布局：左侧“取消”、右侧“确认删除”，默认焦点取消；
- 不要自动删除原始资料目录、SVN 工作副本或用户 `data`。

## 7. 构建、发布和运行 SOP

### 7.1 开发运行

```powershell
cd D:\development\workspace\Codex\自研软件\PengTools\PengToolsV4
python -m pip install -r requirements.txt

```

首次运行时 `data` 可为空；各模块按需创建 JSON。请勿把真实用户 `data` 提交到源码或复制进安装包。

### 7.2 构建私人包

```powershell
cd D:\development\workspace\Codex\自研软件\PengTools\PengToolsV4
.\build_private_release.ps1
```

脚本做以下事情：

1. 记录标准包 `PengToolsHub_Offline_Setup.zip` 的 SHA-256；
2. 从工作区相邻 `02-*` 目录获取生产发版 Excel 模板并更新资源；
3. 清理 `PrivateInstaller` 中上次的 EXE 等旧程序文件，仅保留 `setup.cmd` 和 `README.txt`；
4. 删除 `PrivateInstaller/data`，确保发行包不携带历史用户数据；
5. 使用 PyInstaller 构建 `PengToolsHub_Private.exe`，并打入 QSS、图标、学习种子、发版 Excel 模板等资源；
6. 将 EXE 复制到 `PrivateInstaller`，压缩为 `PengToolsHub_Private_Offline_Setup.zip`；
7. 再次比较标准包哈希，如标准包被修改则直接报错。

### 7.3 标准包维护边界

标准版脚本是 `build_release.ps1`。当前私人需求的规定是“原标准安装包不动”，因此常规维护不要运行它，也不要改标准 ZIP。若未来要把功能正式下放给标准版，应由产品确认后单独评审，并建立与私人版的数据/权限迁移方案。

### 7.4 发布前检查

- [ ] 私人 ZIP 已生成，名称为 `PengToolsHub_Private_Offline_Setup.zip`；
- [ ] 标准 ZIP SHA-256 与构建前一致；
- [ ] ZIP 中没有历史 `data`；
- [ ] 新私人 EXE 可启动；

## 9. 已知限制与维护风险

1. **内网 SVN 不可在外部开发环境验证**：网络、证书、账号、权限均可能造成真实环境差异；代码只能保证本地 SVN 命令调用逻辑。
2. **PyInstaller 单文件启动较慢**：资源会先解压到临时目录；运行时用户数据必须继续从 EXE 旁的 `data` 读取，不能误写到临时目录。
3. **Excel/Word 编辑不是 Office 的完整替代**：学习模块重点是可查看、轻量编辑、复制和导出；复杂宏、公式、批注、样式、嵌入对象可能无法完全保真。
4. **发版模板强约束 23 列表头**：单位模板若改列名/顺序，`release_prep.py` 会报错。更新模板时必须同步评估 `RELEASE_HEADERS` 与写入逻辑。
5. **需求扫描以目录为依据**：扫描会合并已知的本地路径/SVN URL；移动目录、重复路径或手工改 JSON 可能造成重复/错配，应先备份 `requirements.json`。
6. **锁图标不是全量仓库状态视图**：只表示应用成功锁定过的文件；不展示完整 SVN 状态列是为了避免错误状态造成误判。
7. **自我学习属于私人模块**：不要把个人资料、密码或彩蛋机制误合入标准发行包；其中的资料可能含敏感工作信息。

## 12. 交接结论

当前 V4.26 Private 的核心设计目标是：**把日常资料、需求、BUG、SQL、发版清单和日报收敛在一个离线本地工具内，同时把用户操作压到最少。**

后续维护最重要的不是新增页面数量，而是守住以下四条：

1. 用户资料永久保存在本地 `data`，升级不伤数据；
2. 需求、日报、系统配置、SQL、升级准备的数据口径一致；
3. Loading、删除确认、拖动、文件浏览等交互不能妨碍用户继续操作；
4. 标准包不动，私人功能只在私人包中发布，自我学习是唯一可隐藏模块。
