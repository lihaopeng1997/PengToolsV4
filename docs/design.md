# PengTools Hub 产品与技术设计

## 1. 产品定位

PengTools Hub 是面向 Windows 内网办公环境的离线工具工作台。产品参考 uTools 的“主工作台 + 全局悬浮入口”交互，但不包含账号、云同步、插件市场、在线更新、网络请求或遥测。

## 2. 原工具功能盘点

### 统一社会信用代码生成器

- 随机或按省份、区划、机构类型生成代码
- 批量去重生成，可同时模拟企业名称
- 表格展示、选择复制、全部复制、CSV 导出和清空
- 本地校验第 18 位校验字符

### SQL 升级脚本生成工具

- 管理多套系统的 SIM/PROD 数据库连接说明与输出路径模板
- 加载 SQL 文件或粘贴 SQL 文本
- 识别 DDL、DML、混合脚本并拆分
- 从升级 SQL 生成回滚建议，也可从回滚 SQL 生成升级提示
- Oracle 基础检查：括号、引号、INSERT、TRUNCATE、危险操作等
- 为脚本增加系统、环境、日期和操作人表头后导出

### ECIF DOCX 更新工具

- 作为独立离线程序随安装包部署
- PengTools 负责统一入口、存在性检查、启动和错误提示
- 不修改其内部文档规则，避免破坏原程序业务逻辑

## 3. 交互设计

- 启动进入工作台首页，以三张工具卡展示能力与状态
- 左侧导航在工作台、信用代码、SQL 工具之间切换
- `Ctrl+Shift+P` 注册为 Windows 系统级快捷键，在其他程序前台时也能唤出悬浮框
- 悬浮框支持搜索、方向键、回车和鼠标双击启动
- 关闭主窗口后继续驻留系统托盘；托盘可打开主窗口、悬浮框或彻底退出

## 4. 离线与数据边界

- 运行代码中不使用 HTTP、WebSocket、浏览器内核或在线 CDN
- 所有生成、校验和文件处理均在本机完成
- 系统配置存放于本机 `data/systems.json`
- 安装包包含 Python 和 Qt 运行时，内网电脑无需安装 Python
- ECIF 外部程序必须与主程序部署到同一安装目录

## 5. 技术结构

- Python 3.12 + PyQt6：桌面界面和 Windows 事件循环
- Win32 `RegisterHotKey`：系统级快捷键
- `QSystemTrayIcon`：托盘驻留
- PyInstaller：生成离线可执行文件
- ZIP + 安装脚本：无联网、无下载步骤的内网交付

## 6. 验收标准

1. 主程序在无互联网条件下启动并进入工作台。
2. 信用代码可生成、校验、复制和导出。
3. SQL 可分类、拆分、生成反向建议、校验和导出。
4. ECIF 文件存在时可从工作台和悬浮框启动；缺失时有明确提示。
5. 主窗口隐藏后托盘仍可恢复。
6. 在主窗口不处于前台时，`Ctrl+Shift+P` 仍可唤出悬浮框。
7. 安装脚本不受 BOM 或中文控制台编码影响，并能创建桌面快捷方式。
