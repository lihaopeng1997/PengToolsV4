# PengToolsV4 主题预览、弹框图标与格式工具开发需求 V1.0

## 目标

修复设置页主题预览为空白、弹框图标对比度不足的问题；将“加解密”与通用内容格式化分开，形成面向 Java 开发日常的独立“格式工具”模块。所有改动仅适用于 Private 版，不改变本地数据结构、既有 JSON 查看器行为或标准安装包。

## 一、必须修复的视觉问题

### 1. 主题预览卡必须真实显示颜色

**根因**：`ThemeCard.paintEvent()` 由父卡片尝试绘制子控件 `self.preview`，而子控件后续重绘会覆盖绘制结果，因此只剩边框和空白。

**实现要求**：

- 新增一个仅负责预览的 `ThemePreviewWidget(QWidget)`，在它自己的 `paintEvent()` 中绘制；不要再从 `ThemeCard.paintEvent()` 绘制子控件。
- 每张主题卡显示完整微型界面，而不是只有几块色块：左侧导航色、主背景、白色内容卡、一个主按钮、一行弱文本/分隔线。
- 预览必须直接使用 `preview_swatches(theme_id)`，四种内置主题必须明显不同：静谧办公、晴空清晰、暖书房、夜间安读。
- 卡片采用自适应双列布局：设置页宽度足够时两列；窄窗口时一列，不能固定 `112x80` 后留下大面积空白。
- 当前主题需要同时有外框、右上角“当前使用”小标识；不要仅在名称前加字符。
- 添加定向测试：创建四张卡后，预览组件应有非透明像素；四张卡取得的主色和背景色不得相同。

### 2. 所有自定义弹框图标必须清晰

**根因**：`make_badge_label()` 将成功/警告/错误 SVG 染成与 badge 背景同色，视觉对比度近乎为零。

**实现要求**：

- `notice-badge-info/success/warning/error` 的 SVG 使用 `SURFACE`（浅主题白色、夜间主题浅色）或专用 `ON_STATUS` token 渲染，不能继续使用 badge 背景色。
- 关闭弹框的两个选项卡图标也使用明确的前景色：隐藏到托盘使用浅色主色图标，退出使用白色/浅色危险图标；主题切换后仍可读。
- 为 `info/success/warning/error/danger` 建立图标对比度测试，断言 pixmap 非空且前景颜色不同于背景 token。
- 统一替换可替换的原生 `QMessageBox`：`credit_panel.py`、`vin_panel.py`、`docx_panel.py`、`main_window.py`、`settings_panel.py`、`floating_shortcuts_editor.py`。普通提示统一 `show_info/show_warning/show_error`；删除、覆盖、继续等确认统一 `confirm_action`，继续遵守“取消左、确认右、取消默认焦点”。
- `QInputDialog` 可以暂保留，但标题、输入提示和确认/取消顺序应与当前语言一致。

## 二、导航与模块拆分

### 1. 侧栏品牌区

- 侧栏品牌文字只显示 `PengTools` 与一行 `作者：李浩鹏`。
- 移除 `PRIVATE WORKBENCH · Vx` 的常驻展示；版本、构建日期、彩蛋提示移入品牌区 tooltip 或用户菜单的“关于”。
- 不改变彩蛋的密钥、解锁入口和“仅自我学习隐藏”的边界。

### 2. 加解密模块只负责加解密

- 现有导航名称从“加解密”保留为“加解密”。
- 该模块保留：SM2/SM4 参数、请求/响应解密、密文输入、解密结果、现有 `JsonViewer`。
- **加解密中的 JSON 查看和格式化能力保持不变**；这是解密后的主结果呈现，不能迁移或删除。
- 移除该模块内 XML 工作台 Tab 和“送入 XML 工具”按钮；解密结果为 XML 时仅显示一个低干扰的“在格式工具中打开 XML”入口。

### 3. 新增“格式工具”模块

- 在 `开发工具` 分组中新增可见导航项：`格式工具`，图标建议使用通用 code/format 图标，不复用锁或解密图标。
- 增加独立 stack 页面，不能把新模块硬塞入 `GatewayDecodePanel`。
- 同步维护：`main_window.py` 面板创建/stack/导航，`ui/navigation_model.py` 的 `NAV_MODEL`、tooltip、浮动工具栏候选项和图标映射；设置仍处于左下角。
- 旧 `stack_index` 是历史映射，新增项必须使用稳定的新 index，不能改变现有 0-10 的含义。

## 三、格式工具 V1 功能范围

采用三个明确 Tab：`JSON`、`XML`、`SQL`。每个 Tab 使用相同的工作结构：输入区、工具栏、结果区、底部状态，不再堆叠说明卡。

### JSON

- 粘贴、格式化/压缩、校验错误行列、可折叠树、全文搜索并高亮、复制结果、清空。
- 支持带外层双引号、转义的 `\\n`、`\\t`、`\\"` 与 Unicode 转义的常见日志字符串输入。
- 与加解密内 `JsonViewer` 共用纯逻辑或复用组件，但不能影响其现有 API/行为。

### XML

- 复用现有 `XmlWorkspace` 和 `tools/xml_formatter.py` 的已验证能力：粘贴、去外层引号、反转义、格式化、压缩、错误定位、复制、清空。
- 移到格式工具后，保留从加解密页“一键带入 XML”的入口。

### SQL

- 不执行数据库 SQL，只做离线文本整理。
- 支持粘贴/打开文件、格式化缩进、压缩为单行、语句按分号分段、空白 SQL/重复语句统计、复制/导出。
- 重用 `tools/sql_tool.py` 的校验和去重能力；不得把“升级准备”的包生成、系统配置、发版目录混入该页面。
- 校验结果只提示风险（例如空语句、重复、缺少分号、疑似 DDL/DML），明确说明“不能替代 Oracle 实际编译”。

## 四、面向 Java 开发者的后续建议（本轮不实现）

按日常收益排序，后续可以增加但本轮不要扩张：

1. `Java 日志提取`：从多行日志提取异常链、Caused by、请求 traceId，支持复制最短可分享片段。
2. `时间戳转换`：13/10 位时间戳、日期、UTC/北京时间互转。
3. `Base64 / URL / Unicode` 编解码：独立于加解密，不保存敏感内容。
4. `cURL 与 HTTP 报文整理`：请求头、JSON body 语法高亮和脱敏复制；不能联网发请求。
5. `JSON ↔ Java DTO`：仅在用户明确确认模板规则后再做，避免引入不可靠的代码生成器。

## 五、验收与交付

- 在四种主题下实际打开设置页，主题预览必须可见且差异明确。
- 弹出成功、警告、错误、删除确认、退出确认五种对话框，图标均清晰可见；夜间主题同样可读。
- 验证加解密的 JSON 原行为不变；解密到 XML 时可跳转格式工具。
- 验证格式工具 JSON/XML/SQL 三 Tab 的格式化、错误提示和复制；不执行 SQL、不联网、不落盘敏感文本。
- 仅运行本轮相关定向测试；构建 Private 版，标准包不构建、不修改。
- 更新 `.codex_work/grok_handoff/` 摘要，并提交、推送 `origin/main`。

## 六、主题下的命令与代码内容可读性（新增硬性要求）

### 问题

当前 `resources/style.qss` 仍混有大量浅色硬编码，例如 XML 输出区 `#F9FBFF`、JSON 文本区 `#FBFCFF`、运维命令列表 `#34435D`、标题 `#1B2942` 等。主题切换只替换 token 时，这些颜色不会随主题变化；尤其夜间安读主题下，命令示例、SQL/JSON/XML 文本和说明可能与背景对比不足或不可见。

### 实现要求

- 对 `ops-preview`、`ops-command-list`、`sql-input-editor`、`json-text`、`xml-input`、`xml-output`、网关明文区以及新格式工具的全部代码/命令文本区，显式使用主题 token：
  - 背景：`__SURFACE__` 或新增 `__CODE_BG__`
  - 前景：`__TEXT_STRONG__` 或新增 `__CODE_TEXT__`
  - 次级文本：`__TEXT_MUTED__`
  - 选中背景：`__TABLE_SELECT__`
  - 边框：`__BORDER__`
- 如需要代码区与普通输入区有区别，新增 `CODE_BG`、`CODE_TEXT`、`CODE_BORDER`、`CODE_SELECTION` token，并为四套主题分别赋值；禁止再为代码区写固定浅色十六进制值。
- 搜索高亮、错误行、成功状态必须在四套主题下保持可读，不能只依赖背景变化。
- 对 `resources/style.qss` 做一次主题兼容审计：与主题相关的可视文字/背景/边框不得留下硬编码的浅色值；品牌图标资源、危险状态的少量固定色可以保留，但需在夜间主题下验证对比度。
- 增加离屏定向测试：依次应用 calm / clear / warm / night，读取上述代码区控件的调色板或渲染截图，断言前景与背景不同且文本区不是透明/空白。
