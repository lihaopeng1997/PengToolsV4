# PengToolsV4 接口监听与需求文件库综合优化需求 V1.0

> 本文为 Grok 开发文档。本轮只改 Private 版，标准安装包保持不变；用户本地数据必须兼容。已完成的夜间主题与响应式布局作为基线，不要回退。

## 一、问题与目标

### 1. 接口监听

当前存在“监听成功，但请求列表和接口详情没有内容”的关键问题。目标是学习 Fiddler2、mitmproxy、Charles、HTTP Toolkit 的本机代理模式：软件打开后点击一次监听，即可看到本机浏览器/IE 的接口调用，不要求用户先选择浏览器或手动开启 CDP。

### 2. 需求文件管理

需求管理中的“文件与版本库”内容需要成为真正可用的文件库：文件可排序、实时搜索、拼音搜索、全部展开/折叠、列宽拖动、完整路径可查看，树节点在窄窗口下不能只显示半截文字。

### 3. 模块命名与联动

- “SQL 脚本”不是独立的最终业务模块，而是发版准备的输入材料。
- 原“SQL 整理”模块重命名为“发版联动”。
- 原“文件与版本库”页签重命名为“文件库”。
- 发版联动中继续保留 SQL 编辑、校验、按系统拆分、关联需求/BUG、生成发版 Excel 等能力。

### 4. 拼音搜索

可以实现。建议使用 `pypinyin` 生成中文字段的全拼和首字母索引；若不希望新增依赖，可实现常用汉字首字母表作为降级方案。索引只保存文件名、标题、系统、分类等非敏感元数据，不保存报文和密钥。

## 二、接口监听：Fiddler 式开箱即用方案

### 2.1 默认入口

接口排查页面打开后，顶部只保留：

- `开始监听`（主按钮）
- `停止监听`
- `监听模式` 状态胶囊
- `捕获数 / 最近请求时间`
- `更多`（高级 CDP、证书、端口、过滤设置）

不要把“选择浏览器 EXE”和“调试端口”放在首次使用的主流程中。

用户流程：

```text
打开软件 → 接口排查 → 开始监听 → 正常操作 Chrome/Edge/IE → 自动出现请求
```

### 2.2 技术模式

#### 默认：本机通用代理

- 内部启动 mitmproxy 或同等 MITM 组件，绑定 `127.0.0.1`。
- 临时备份当前用户 WinINet/Windows 代理，再切换到 PengTools 本地代理。
- 停止监听、正常退出、异常恢复时还原原代理。
- HTTPS 首次使用弹出明确确认，安装当前用户证书；提供移除证书入口。
- 不允许静默修改代理，不允许绑定 `0.0.0.0`、局域网或公网。
- 支持 Chrome、Edge、360、QQ、搜狗、Brave、Opera、IE 以及遵循系统代理的程序。
- 软件关闭后不应残留代理；启动时发现上次未恢复，提供“立即恢复原代理”。

#### 高级：Chromium CDP

- 保留自动识别、手动选择浏览器 EXE、启动独立资料目录、连接已有 `127.0.0.1` 调试端口。
- 只作为高级模式，不强制用户选择浏览器。
- 不打包、不内置任何浏览器 EXE 或运行时。
- CDP WebSocket 必须绕过 HTTP_PROXY/HTTPS_PROXY；连接失败要展示真实错误。

#### 兼容：IE

- 复用本机代理模式，不要求用户寻找 IE 的特殊入口。
- IE 模式下隐藏 Chromium 专用控件；仅展示代理、证书和恢复状态。

### 2.3 “监听成功但没有请求”修复验收

必须修复整个事件链路，而不是只改按钮文案：

1. 只有代理线程已启动、端口可连接、事件回调已注册，才能显示“监听成功”。
2. 启动后 10 秒没有流量时显示“监听已建立，等待浏览器请求”，不能误报失败，也不能伪装有数据。
3. 代理/CDP 断开时状态改为“已断开”，停止显示监听中。
4. 所有后台抓包线程通过 Qt signal/slot 或线程安全队列通知 UI，禁止后台线程直接操作 QWidget。
5. CDP 必须覆盖：`Network.enable`、`requestWillBeSent`、`responseReceived`、`loadingFinished`、`loadingFailed`、重定向和空响应。
6. 代理 flow 必须在请求阶段就创建记录，即使响应失败或无响应体也要显示 URL、方法、请求头和失败原因。
7. 使用稳定 requestId 合并请求/响应；重定向不能覆盖原记录。
8. 默认筛选不得隐藏所有业务请求：默认保留 XHR/Fetch、Document、未知类型；静态资源可默认关闭但可一键显示。
9. 第一条请求到达后实时更新列表和计数；新请求不能覆盖当前选中的详情。
10. 响应体获取失败时保留请求摘要，并明确提示“响应体暂不可获取”。
11. 每个监听模式提供“测试监听”按钮，访问一个本机最小测试地址验证事件链路；不能向业务系统发送请求。

### 2.4 Fiddler 式界面

左侧请求列表：

- 状态、方法、host/path、耗时、类型、时间为默认列。
- 大小、来源、失败原因为可选列。
- 支持实时搜索 URL、host、path、method、状态码、Content-Type。
- 支持全部、XHR/Fetch、失败、慢请求、JSON/XML、静态资源筛选。
- 显示“当前显示 / 总捕获”。

右侧详情：

- 概览：URL、方法、状态、耗时、来源、大小、失败原因。
- 请求：Query、Headers、Cookie、Body。
- 响应：Headers、Body。
- 验证草稿：本地地址重写、Postman v2.1、cURL。
- JSON/XML 自动格式化，可送入格式工具和加解密。
- Cookie、Authorization、Token 默认脱敏；复制或显示敏感内容前提示。
- 明确显示“只生成草稿，不实际发送请求”。

### 2.5 安全与数据

- 报文、Cookie、Token、密钥只存内存。
- 停止监听、切换会话、退出应用后清空。
- 配置文件只能保存浏览器路径、端口、代理偏好、筛选和列宽，不保存报文。
- 不增加云端服务、外网服务或业务请求发送能力。

## 三、需求管理：文件库

### 3.1 命名

- 需求管理右侧页签：`文件与版本库` → `文件库`。
- 页面副标题：`需求、BUG、SQL、通知、接口文档与版本库文件`。
- SVN 状态可继续以锁图标展示，不恢复为大段状态文字。

### 3.2 文件库列表

完整展示当前需求目录中的文件和文件夹，不只展示附件摘要。字段：

- 名称
- 类型（文件夹、Excel、Word、TXT、SQL、JSON、XML、图片、其他）
- 修改时间
- 大小
- SVN/本地路径
- 锁定图标（如有）

支持：

- 点击列头排序；名称、修改时间、类型、大小、路径均可升降序。
- 当前列表实时搜索，不重新扫描目录；搜索结果保留树层级路径。
- 搜索范围：文件名、相对路径、类型、扩展名、需求标题、BUG 编号、系统名。
- 双击文件打开预览/编辑器；右键打开文件夹、复制路径、刷新、导出。
- 多选文件后复制、导出或删除；删除继续二次确认，默认焦点为取消。
- 文件列表和详情区使用可拖拽 splitter；列边界明显可拖动。

### 3.3 树结构

- 顶层日期/月份始终完整展示并加深字体。
- 月份、分类、需求/BUG、文件夹、文件使用不同字号和颜色层级。
- 提供“全部展开”和“全部折叠”按钮，位于树工具栏，与全选操作同一行。
- 折叠/展开状态按目录保存到本地。
- 搜索命中时自动展开父节点并高亮命中项；清除搜索后恢复用户原来的展开状态。
- 左侧树支持拖动 splitter，不允许拖动标题文本造成节点消失。

### 3.4 窄窗口文字策略

禁止出现“名称展示一半”的状态，按以下优先级处理：

1. 第一优先：树节点提供 Tooltip 展示完整名称和路径。
2. 第二优先：需求/BUG 标题允许两行换行，行高自适应。
3. 第三优先：仅对非关键描述省略为 `…`，需求号、BUG 号、日期、文件名不能截断到无法识别。
4. 若宽度不足，自动增高节点，不要把文字压缩成半截；必要时把完整名称放到右侧详情。
5. 用户手动拖拽树宽度后，短期内不要自动覆盖其宽度。

## 四、拼音搜索

### 4.1 支持范围

需求管理、文件库、版本库、系统配置、学习资料和日报标题的搜索统一支持：

- 中文原文：`车险承保`
- 全拼：`chexian chengbao` 或 `chexianchengbao`
- 首字母：`cxcb`
- 混合：`cx 承保`、`REQ 车险`
- 英文和数字：REQ 编号、BUG 编号、SVN 路径和扩展名

### 4.2 实现建议

- 优先使用 `pypinyin` 生成 `pinyin_full` 与 `pinyin_initials`。
- 首次扫描或导入时建立内存索引，目录内容变化后增量更新。
- 如果安装包不希望增加依赖，提供内置常用汉字首字母降级表；不要因为无法生成拼音而阻塞原文搜索。
- 搜索索引只存非敏感元数据，不存报文、密码、Key 或 Cookie。
- 150ms 防抖实时搜索；命中项高亮并自动定位。
- 重音符号、大小写、空格、连字符统一归一化。

## 五、SQL 脚本改为发版联动

### 5.1 模块命名

- “SQL 整理”页签和导航名称改为“发版联动”。
- 旧数据 key、信号名和数据文件名保持兼容，避免升级丢数据；仅修改 UI 展示名称。
- 旧入口跳转到发版联动，不保留重复的“SQL 整理”大标题。

### 5.2 发版联动职责

发版联动不是单纯 SQL 编辑器，而是将需求/BUG、系统、SQL 和发版 Excel 串起来：

- 选择当前系统，可直接从系统配置下拉选择。
- 从需求管理勾选本次发版需求和 BUG。
- 按系统筛选并生成对应 SQL，不能把不同系统脚本混在一起。
- 支持粘贴、导入、格式化、检查、复制和导出 SQL。
- 显示 SQL 来源、需求编号、系统、是否需要执行、是否已纳入本次发版。
- 选择发版日期后，自动匹配或新建模板 Sheet，保留既有表头、行宽和列宽。
- 生成发版 Excel 时带入分支名、任务编码、任务内容、升级公告、SQL、周边系统通知、临时升级等字段。
- 需求中没有开发分支时允许保存，发版时分支字段留空并提示，不阻断其他内容保存。

## 六、验收标准

### 接口监听

- 不选择浏览器，点击一次开始监听，Chrome、Edge、IE 操作页面后能看到请求。
- 监听状态、请求计数、最近请求时间准确。
- 请求列表至少显示一条业务请求；请求/响应详情可打开。
- 无请求时显示等待状态，不显示虚假成功。
- 代理停止、异常退出、证书恢复正确。

### 文件库

- 文件和文件夹按名称、时间、类型、大小排序。
- 当前列表可实时搜索，中文原文、全拼、首字母、编号和扩展名均可命中。
- 全部展开/折叠有效。
- 列宽可拖拽，树宽可拖拽；窄窗口下名称不显示半截。
- 右键打开文件夹、复制路径、导出、删除确认有效。

### 发版联动

- 所有旧“SQL 整理”入口显示为“发版联动”。
- 需求、BUG、系统、SQL、发版 Excel 之间的联动不丢失。
- 多系统时分别生成对应 SQL。

### 加解密

- Key、IV/Nonce、算法、模式、编码参数清晰可见并可录入。
- 接口报文送入时不覆盖已有密钥参数。

只运行本轮接口监听、文件库、拼音搜索、发版联动和加解密定向测试；构建并启动 Private 版，更新 `.codex_work/grok_handoff/`，提交并推送 GitHub main。标准安装包不得修改。
