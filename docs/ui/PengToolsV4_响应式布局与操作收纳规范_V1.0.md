# PengToolsV4 响应式布局与操作收纳规范 V1.0

## 目标

把现有“缩小窗口后控件被挤压”的行为，改为苹果官网式的渐进响应：宽屏完整信息，中屏收敛说明，窄屏保留主操作，其余操作进入“更多”。不隐藏业务数据，不通过缩小字体解决拥挤。

## 全局断点

| 模式 | 主窗口宽度 | 导航 | 页面内容 |
|---|---:|---|---|
| Wide | >=1440 | 248px，图标+文字 | 完整标题、完整按钮、双/三列布局 |
| Standard | 1280-1439 | 220px，图标+文字 | 收紧间距，保留文字按钮 |
| Compact | 1080-1279 | 84px，仅图标 | 次要说明隐藏；次要按钮进入“更多” |
| Narrow | 960-1079 | 72px，仅图标 | 单列主体；每组只保留一个主按钮，其余进入“更多” |

- 高度低于 720px：页面标题副标题隐藏；只保留标题、状态与主操作；卡片间距从 16px 收紧为 10px。
- 最小尺寸仍为 960×640，禁止继续压缩到无法操作。
- 用统一 `ResponsiveActionBar` 管理“可见按钮 + 更多菜单”，按钮必须是同一个 QAction，不能出现两个不同入口逻辑。

## 按钮规则

- Wide/Standard：显示全部常规操作，危险删除始终独立且靠右。
- Compact：显示主操作、刷新/搜索、更多；导入、帮助、配置、低频批量操作进入更多。
- Narrow：显示主操作和更多，图标按钮必须有 tooltip；文字按钮不允许折行或半截显示。
- 长任务按钮保持可见，Loading 仍为独立浮层，不推动按钮位置。

## 模块响应细则

### 首页

- Wide/Standard：最近需求与待升级事项两列，各最多 5 条。
- Compact/Narrow：两张卡纵向排列，各最多 3 条；“全部/升级准备”保留在卡头右侧图标入口。
- 常用工具：Wide/Standard 显示图标+文字；Compact 显示图标+短文字；Narrow 只显示前 4 个图标，其余在“更多工具”。

### 需求管理

- Wide/Standard：左树与右详情按可拖动 splitter 展示。
- Compact：左树默认 280px；右侧详情优先保证 520px；SVN 工具栏仅显示打开目录、刷新、提交、更多。
- Narrow：左树与右详情仍可拖动，不允许任意侧折叠为 0；文件/SVN、SQL 脚本、发布与联动 Tab 内按钮收纳到更多。
- 完成标记继续按现有 >=520 双列 / <520 单列规则重排。

### 升级准备 / SQL

- Wide：升级筛选、选择、生成完整展示；SQL 输入头显示导入、粘贴、清空。
- Compact：升级上下文默认折叠；SQL 只保留导入、粘贴、更多，清空进入更多。
- Narrow：系统选择、环境、日期保留；输出目录单独一行；检查 SQL 与生成预览进入更多，导出全部保持主按钮。

### 接口文档、格式工具、接口排查、加解密

- 双栏编辑器在 Compact 以下改为上下分栏，用户拖动的 splitter 尺寸按模式分别保存。
- 文本编辑器不能低于 180px；详情区不能被按钮挤掉。
- 接口排查 Narrow 模式：顶部仅显示连接状态、开始/停止监听、更多；请求详情改为 Tab（请求/响应/验证草稿）。

### 设置、学习、日报、运维、证件、VIN

- 设置主题卡：Wide/Standard 两列，Compact/Narrow 单列。
- 学习/日报/运维的左右列表改为纵向或保持 splitter，不能因窗口缩小隐藏正文编辑区。
- 表格列优先级：名称/标题、状态、时间始终展示；类型、大小、路径等次级列允许隐藏并在“列设置”恢复。

## 技术实现

- 扩展 `ui/responsive.py`，保留现有四档断点，新增 `ActionDensity`（full/compact/overflow）和统一 action-bar helper。
- 每个面板实现 `apply_layout_mode(mode, low_height)`，实际重排 layout、设置按钮可见性/菜单项，不能只保存 mode。
- 所有折叠/重排在窗口 resize 后 120ms 防抖执行；避免循环 resize。
- 通过 objectName/property 统一 QSS：`responsive-hidden`、`responsive-overflow`、`responsive-icon-only`；主题不改变断点逻辑。
- 用户主动拖动的 splitter 和表格列宽继续优先；模式变化只提供安全最小宽度，不覆盖用户在同一模式下的选择。

## 验收

- 在 1440×900、1280×800、1080×720、960×640 四个尺寸逐页验证。
- 任意按钮不得半截、重叠、折行或因缩放消失；隐藏的次要操作必须能从“更多”找到。
- 主操作始终可见：导出、生成、保存、提交、开始/停止监听、删除确认入口。
- 验证需求 splitter、文件列宽和编辑器 splitter 在同一模式下可恢复。
- 仅运行 UI/响应式相关定向测试，构建 Private 版并写 handoff 摘要后提交推送。
