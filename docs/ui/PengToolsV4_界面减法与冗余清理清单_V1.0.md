# PengToolsV4 界面减法与冗余清理清单

> 目标：让页面安静、干净、层级明确。  
> 原则：删掉重复表达，不删业务能力；同一信息在同一屏只出现一次，除非第二次展示承担不同决策职责。

## 1. 全局减法规则

1. **一个页面只保留一个页面标题。** 页面标题区已说明模块用途后，正文第一张卡不得再次使用同名大标题。
2. **一个 Tab 不重复自己的名字。** Tab 名已为“文件与 SVN”，Tab 内容里不再放一个“文件与 SVN”区块标题；只保留路径、状态或实际操作。
3. **说明文字最多一层。** 页面副标题、区域提示、字段提示三者不能同时解释同一件事。优先级：字段级提示 > 区域一句话 > 页面副标题。
4. **卡片整块可点击时，不再放重复的长 CTA。** 卡片标题、描述和“打开 XX”按钮三次表达同一个入口时，保留标题 + 一句结果描述 + 小箭头即可。
5. **状态不重复。** 当前系统、上线状态、文件数量、离线状态等，在一屏只保留最接近用户决策的位置；其它位置只展示有变化时的差异。
6. **路径只在需要操作时完整展示。** 页面摘要不展示长 SVN/本地路径；在文件/路径工具栏展示可复制的完整路径即可。
7. **所有“操作说明”默认不占版面。** 可以使用 tooltip、空状态说明、首次使用提示或“？”帮助，不要长期占据卡片高度。
8. **每一个可见标签都必须回答一个问题。** 它要么说明“这是什么”、要么说明“当前状态”、要么说明“下一步怎么做”；三者都不回答的文字删除。

## 2. 必改清单

### 2.1 首页工作台：从模块陈列改为今天工作

**当前冗余/问题**

- 标题、副标题、离线胶囊、三张工具卡、卡片描述、卡片 CTA、底部快捷键提示同时出现，信息很多但没有“你接下来该做什么”。
- 三张卡中的标题、描述、按钮反复表达“打开某模块”。
- 用户已明确首页优先级是“最近需求和待升级事项”，当前三卡方案不符合。

**改造**

1. 首页标题改为“工作台”，副标题只保留日期/一句轻问候，例如“今天先处理最近的交付事项”。不要显示“开发工具工作台”。
2. 第一行改为两张任务卡：
   - 左：`最近需求`，最多 5 条，显示标题、系统、最近修改时间、状态；点击进入对应需求。
   - 右：`待升级事项`，按本周/下周/未定分组，显示需求数和最近升级日期；点击进入升级准备。
3. “加解密、证件、接口文档、VIN、运维”等工具统一放到底部紧凑“常用工具”行，仅图标 + 文字；不再使用大卡片。
4. 删除工具卡内的长按钮文字；如果保留卡片入口，只使用右下角箭头。
5. 离线状态移入标题右侧 6px 状态点 + `本地工作`短文字；删除大胶囊和底部快捷键常驻提示。

### 2.2 需求管理：去掉 Tab 内重复标题与常驻说明

**当前冗余/问题**

- 页面副标题已说明“需求、BUG、文件、SQL 与升级联动”，工具栏下的 `svn_activity` 又长期说明“右侧用文件与 SVN / 关联 SQL / 上线与联动完成工作”。
- `文件与 SVN`、`关联 SQL`、`上线与联动`既是 Tab 名，又在 Tab 内部再次作为 `zone-title` 显示。
- 右侧详情卡里“需求详情”与具体需求标题共存，选中项后信息层级可进一步收紧。
- “上线与联动”内的提示文字重复说明 Tab 的作用。

**改造**

1. 删除常驻 `svn_activity` 说明；它只在扫描/SVN 任务运行时显示为小型状态提示。
2. 删除每个 Tab 内与 Tab 同名的 `zone-title`：
   - 文件 Tab 顶部直接从路径工具栏开始；
   - SQL Tab 直接从来源列表/预览开始；
   - 联动 Tab 直接从操作卡开始。
3. 详情摘要卡标题只显示实际需求标题；`需求详情`降为 11px 眉标，或在已有详情语境下完全删除。
4. 路径只在文件 Tab 中展示，摘要卡只显示“已绑定资料 / 未绑定资料”状态与一个打开/复制图标。
5. 左树“需求目录 + 数量”保留；工具栏只保留勾选、删除、展开、折叠，不加解释性文字。
6. 关联 SQL Tab 不再同时显示“关联 SQL”标题、Tab 名和无 SQL 空说明；空状态只保留一枚图标、一句“暂未关联 SQL”和“添加 SQL”动作。

### 2.3 升级准备 / SQL：合并多层标题和懒人流程说明

**当前冗余/问题**

- “懒人流程”长说明、“发版路径与筛选”、“候选需求 / BUG”、“生成”、“额外 SQL 归属”和多条 hint 连续出现，用户先读说明再做事。
- SQL 整理页存在“SQL 输入来源 / 交付信息 / 编辑与预览 / 输入 / 预览 / 检查 / 导出”多层标题，视觉切分过度。

**改造**

1. 升级准备顶部只保留一行简化流程：`选择日期 → 勾选事项 → 生成资料`，做成 3 个弱步骤，不再显示长段落。
2. 将“生产发版 SVN、发版清单目录、升级日期、系统筛选”放入单个可折叠的“升级上下文”条；默认仅展示日期、系统和候选数量。
3. 候选列表上只保留数量和筛选；删除“候选需求 / BUG”重复标题。
4. 生成区域只保留“生成升级资料”主标题；“额外 SQL 归属”作为字段 label，SQL 汇总规则放 tooltip。
5. SQL 整理 Tab 合并为三块：`输入`、`预览`、`导出`。删除“SQL 输入来源”“编辑与预览”“检查 / 导出”等外层重复标题。
6. 路径提示仅在路径为空或错误时显示；有效路径不再长期显示解释文字。

### 2.4 加解密：参数、模式、输入输出不要三重说明

**当前冗余/问题**

- 页面标题副标题、离线胶囊、固定系统说明、环境/密钥参数、Tab、输入区/输出区标题、底部说明同时出现。
- “新车险系统（固定兼容模式）”是固定信息，不需要占用一行表单字段。

**改造**

1. 页面副标题明确“本地处理，内容不保存”即可；删除或缩小重复离线胶囊。
2. 固定系统兼容信息移入 tooltip/帮助，不再占参数区一行。
3. 参数区收为默认折叠的“解密参数”，展开后才显示环境、密钥等字段；用户主要任务是粘贴、解密、查看结果。
4. Tab 内标题只保留“输入”和“结果”，不要重复“国密解密”“XML 工作台”等 Tab 名。
5. 格式/错误统计放编辑器底部状态栏，不在正文另放说明卡。

### 2.5 接口文档更新：配置项、文件浏览、日志说明收敛

**当前冗余/问题**

- 多个路径字段前后都有 label、状态、路径说明；文件浏览、SQL 输入、日志又各有标题和提示。

**改造**

1. 顶部改为一行“源文档 / 输出目录 / 日期”，字段内 placeholder 承担示例；删除重复路径说明。
2. 模板匹配状态只在匹配异常时显示；正常状态不占行。
3. 左侧文件列表标题使用当前文件夹名，不再显示泛化“输出文件浏览”。
4. 日志默认折叠为底部 32px 状态条；仅在运行或失败时展开，避免常驻大面积空日志。

### 2.6 设置：减少“标题 + 说明 + 分组标题 + 字段标签”的层数

**当前冗余/问题**

- 页面标题/副标题、外观 GroupBox、主题标题、主题提示、主题备注形成四层介绍。
- 关闭行为的选项、当前状态说明、安全提醒三处文字有部分重复。

**改造**

1. 设置页面标题保留；副标题删掉或改为极短“个人偏好”。
2. “外观”直接作为 GroupBox 标题，删除内部重复的“大标题式主题标题”；保留一句主题提示即可，删除主题备注或移入 tooltip。
3. 主题卡下方不再重复“主题不影响数据”等常驻说明，改为标题右侧 `?` 帮助。
4. 关闭行为只保留当前选择和一条动态状态提示；安全提醒只对“直接退出”显示。
5. 浮动工具栏设置移到独立“快捷入口”分组，避免和透明度、置顶、启动显示混在一起。

### 2.7 日报与自我学习：状态只在相关模式下出现

1. Word 编辑器状态只在当前打开 Word 条目时显示，不能在 Excel/TXT/Markdown 浏览时常驻。
2. “资料库/知识与资料/当前条目标题/类型徽标”中，资料库为左栏语境，右侧只保留实际条目标题和必要类型徽标。
3. 日报页“提醒时间 / 到点通过托盘提醒”迁入设置；日报编辑页只显示当前提醒已开启/未开启的小链接。
4. Excel 工具操作合并为两个下拉：`复制`（单元格/整行/当前展示/整表）和 `导出`（当前展示/整表），不同时展示一排重复动作。

### 2.8 运维助手、证件、VIN：清理长说明和重复结果标签

1. 运维助手将安全说明缩为首次打开可关闭的横条；风险等级保留在命令详情头部，不重复于警告/说明/输出区。
2. 运维详情只保留“说明、命令、结果”三段；“使用指南、命令预览、输出说明”等泛化标题删除或合并。
3. 证件和 VIN 页面保持“参数 → 结果”两段。`格式说明`仅在用户切换类型或输入非法时展示；结果数量放表格右上，不在底部再重复。

## 3. 实施守则

- 删除 UI 元素前确认其不承担状态、错误、权限或数据来源职责。
- 同类信息优先用图标/徽标替换长句，但重要风险信息不得只靠图标。
- 任何删除要同时清理关联的空 layout、stretch、objectName、翻译文案和测试断言。
- 不得通过缩小字号、降低透明度来“隐藏冗余”；应该真正移除或按条件显示。
- 只改 UI 呈现层，不改变需求—SQL—升级—日报的业务联动、不改变本地数据结构。

## 4. 验收标准

1. 每页首屏最多包含：页面标题区、一个筛选/上下文区、一个主体区；不出现三层以上连续标题。
2. 用户切换任一需求右侧 Tab 时，不会看到 Tab 名与内部主标题重复。
3. 首页首屏明确展示最近需求和待升级事项，不再以多个“打开模块”大卡片为主。
4. 升级准备、SQL 整理、加解密、设置的常驻说明文字明显减少，但首次使用仍能通过 placeholder、tooltip、空状态理解操作。
5. 960×640 下不通过隐藏关键信息解决拥挤，而是先移除冗余、折叠次级说明。
