# PengToolsV4 企业级界面全量布局蓝图 V2.0

> 日期：2026-07-20  
> 产品：PengToolsV4 Private  
> 设计代号：Astra Workbench  
> 实现技术：Python 3.12 + PyQt6 + 本地 SVG + QSS  
> 本文是像素级 UI 实施标准。与 V1.0 冲突时，布局、颜色、响应式和视觉规范以本文为准；业务、数据及安全边界继续遵守 V1.0 与 `AGENTS.md`。

---

## 1. 设计结论

现有界面不舒服的主要原因不是某个颜色，而是五类问题叠加：

1. 左侧导航大面积深色，文字、图标和背景对比不稳定，视觉重量过大。
2. 页面标题、筛选、按钮、主体卡片缺少固定栅格，不同模块的起始位置和留白不一致。
3. 按钮按“代码出现顺序”摆放，没有按主任务、次任务、风险任务分组。
4. 窗口缩小时依赖控件自然挤压，导致文字截断、按钮换行、下拉框难以识别。
5. 科技感主要来自深色和渐变，而不是来自精细状态、信息密度、图标、动效和结构。

V2.0 采用“浅色企业工作台 + 冷色科技细节”：主界面保持明亮、清晰、长时间使用不疲劳；科技感用细线、轻微渐变、状态光点、数据徽标、精准 hover 和流畅布局体现。

### 3.2 材质规则

- 页面背景只使用极浅冷灰，不使用大面积渐变。

- 全局圆角：输入/按钮 8px，卡片 12px，弹窗 16px，状态胶囊 999px。
- 所有文字对比达到正常办公可读性；浅色导航正文不得低于 `#445168`。

## 5. 左侧导航完整重设计

### 5.1 标准宽屏状态（窗口宽度 ≥ 1280）

- 导航宽度：248px。
- 背景：`#F7F9FC`；右边 1px `#DDE4EF`。
- 内边距：左右 12px，上 14px，下 12px。
- 品牌区：高度 64px，不使用深色卡片；左侧 36px 应用图标，右侧两行：`PengTools` 18px/600，`PRIVATE WORKBENCH · V4.27` 11px/500。
- 构建日期不常驻占一行，移入版本 tooltip；减少品牌区高度。
- 分组标题：距品牌区 14px，左边 12px，字号 11px，字距轻微增加，颜色 `#8A95A9`。
- 导航项：高 42px，左右内边距 12px，圆角 9px，图标 20px，图标与文字间 12px。
- 普通态：透明背景，文字 `#445168`，图标 `#68768F`。
- hover：`#EEF2F8`，文字 `#26344C`。
- 激活：`#E9EDFF`，文字/图标 `#4056CF`；左侧距边缘 3px 位置显示 3×22px 主色圆角指示条。
- 导航项之间 2px，不加分割线。
- 日报右侧可显示 8px 提醒圆点；不得用高饱和数字徽标干扰。
- 自我学习解锁后放在“个人效率”分组，不插入业务工具组中间。

### 5.2 导航分组和顺序

```text
工作台
  首页

交付管理
  需求管理
  升级准备
  接口文档更新
  日报

开发工具
  加解密
  证件类型
  车辆 VIN
  运维助手

个人效率（解锁后）
  自我学习

底部固定
  设置
```

必须调整当前“按历史 index 排列”的视觉顺序，但可保留现有 stack index 映射；增加显式导航模型，禁止再依靠视觉顺序等同数组下标。

### 5.3 底部区域

- 设置固定在底部，和主导航间保留弹性空间与 1px 分隔线。
- 作者、语言、快捷键不再各占一行。标准状态只显示：设置项、右下角 28px 用户圆形标识 `LH`、当前语言缩写。
- 点击用户标识弹出轻量菜单：版本、语言、快捷键、关于、退出。
- `Ctrl + Shift + P` 移入 tooltip/用户菜单；不再常驻一个胶囊占空间。

### 5.4 响应式折叠

| 窗口宽度 | 导航宽度 | 展示 | 隐藏/迁移 |
|---:|---:|---|---|
| ≥ 1440 | 248px | 图标、文字、分组、品牌完整 | 无 |
| 1280–1439 | 220px | 图标、文字、品牌完整 | 版本副标题缩短 |
| 1080–1279 | 84px | 图标、应用标识、激活条 | 隐藏导航文字、分组标题；tooltip 显示全称 |
| 960–1079 | 72px | 仅 18px 图标和激活条 | 用户区仅头像；语言/版本移入菜单 |

导航折叠后不得简单设置固定宽度压缩文字。必须进入明确的 `compactNavigation` 状态，隐藏文字控件、居中图标、调整左右 padding，并为每项提供 tooltip。

高度小于 720px 时：导航列表进入独立滚动区；品牌区降为 48px；隐藏版本副标题；底部只保留设置与用户标识。设置始终可达。

## 6. 页面通用骨架

### 6.1 标题区

- 高度：标准 72px，紧凑 60px。
- 左侧：32px 模块图标底板（浅模块色）+ 标题和一句副标题。
- 图标与标题组间距 12px；标题与副标题间距 2px。
- 右侧：只放页面级状态和一个 Primary 主操作；最多再放一个 `...` 更多按钮。
- 页面标题左边与主体卡片左边严格对齐。
- 页面内部不得再出现重复标题，如顶层“需求管理”后又出现同样的大卡片标题。

### 6.2 工具/筛选区

- 高度：48px；复杂筛选最大 64px。
- 背景白色，圆角 12px，边框 1px；内部水平 padding 12px。
- 搜索框优先占剩余空间，建议 320–520px；下拉控件标准 140px，短枚举 112px。
- 主按钮不放筛选区内；筛选区只放搜索、过滤、排序、清除。
- 过滤项超过可用宽度时：先缩短搜索框到 240px，再将低频筛选收入“更多筛选”，不得换成两行挤压主体。

### 6.3 主体区

- 数据页面使用全部剩余高度，避免外层无意义滚动。
- 表单页面使用 `QScrollArea`，内容最大 1180px 居中；卡片宽度自适应。
- Tree/Table/Editor 都必须在卡片内充满剩余区域；标题栏固定、内容滚动。
- 分隔条标准 8px，可视抓手 2px；hover 变主色并将鼠标改为 ResizeCursor。

### 6.4 状态栏

底部状态栏高度 28px，背景 `#FAFBFD`，上边 1px；左侧为当前页面状态，右侧仅显示时钟、离线标记或任务状态。普通成功消息 3秒后消失，不作为永久说明区。

## 8. 各模块精确布局

## 8.1 首页工作台

### 宽屏

- 标题区右侧显示“离线运行”状态点，不放大胶囊。
- 第一行：3 个核心快捷卡片，需求管理 / 升级准备 / 加解密，每个最小宽 260px，高 132px。
- 第二行：近期工作区，左侧 2/3 显示“今日待办/最近需求”，右侧 1/3 显示“日报提醒/最近使用工具”。
- 工具入口不再使用 2×3 同权大卡片；低频工具放成一行紧凑入口。

### 缩放

- ≥1360：3列核心卡片。
- 1100–1359：2列，第三张换到下一行；近期区 60/40。
- <1100：单列卡片，隐藏卡片长描述，只保留标题、一句状态和箭头。

## 8.2 需求管理

### 标题与动作

- 标题左：“需求管理”+“需求、BUG、文件、SQL 与升级联动”。
- 右侧：次按钮“扫描目录”，Primary“新增需求”；系统配置、导入、更新全部移入 `...` 菜单或次级动作区。
- SVN 正在执行时，标题右侧出现非挤压任务浮层，不改变按钮位置。

### 筛选区

顺序：搜索框（自适应）→ 系统 150px → 类型 112px → 状态 128px → 排序 130px → 清除筛选图标。

<1180px 时隐藏“排序”到更多筛选；<1060px 时类型与状态合并进入筛选弹层，只保留搜索和系统。

### 主体分栏

- 宽屏默认左 330px / 右剩余，比例约 28/72。
- 左侧最小 260px，最大 480px；右侧最小 520px。
- 紧凑导航下仍优先保证右侧 520px。
- 960px 最窄模式：左侧 300px，右侧最小 560px，允许主工作区横向滚动作为最后兜底；不得将右侧表格列压成不可读。

### 左侧目录卡片

- 顶部 44px：标题“需求目录” + `当前/总数`。
- 工具栏 40px 单行：16px 勾选框“全选”｜删除图标按钮；右侧为展开、折叠纯图标按钮，tooltip 显示名称。
- 宽度 <300px 时，隐藏“全选”文字，仅保留勾选框；删除、展开、折叠仍显示。
- 月份行：完整 `2026年07月`，右侧数量；不得省略顶层日期。
- 子项第一行：编号/BUG号主色 12px/600；标题 13px/500。
- 子项第二行：系统名、修改日期、SQL/周边通知/临时升级微标记。空间不足时先隐藏修改日期，再隐藏低优先标记文字，只保留图标。
- 拖动时显示清晰插入线/目标月份背景；drop 完成后选中移动项并滚动可见。

### 右侧详情

- 顶部摘要卡 112px：第一行标题、编辑按钮、状态徽标；第二行展示系统、计划上线、实际上线、路径状态。长路径不放这里。
- 摘要卡下为水平 Tabs：`文件与 SVN`、`关联 SQL`、`上线与联动`；避免文件和 SQL 永久上下挤在同一视区。默认进入“文件与 SVN”。
- 文件 Tab：上方 40px 路径工具栏，下方表格充满；SVN 动作放表格上方次级工具栏，分组为“浏览｜版本控制｜提交”。
- 关联 SQL Tab：左 42% SQL 文件/来源列表，右 58% 预览；可拖动。
- 上线与联动 Tab：计划/实际日期、是否通知周边、是否临时升级、日报联动和进入升级准备。
- 这样替换当前“详情卡 + 文件区 + SVN 8按钮 + SQL区”的纵向堆叠，减少按钮海和高度争抢。

### 右侧缩放规则

- ≥1250 内容宽：摘要字段一行 4列。
- 1050–1249 内容宽：摘要字段两行 2列；路径只显示图标状态。
- <1050 内容宽：详情摘要只保留标题、状态、系统、上线日期；其它进入“更多信息”抽屉。
- 文件表隐藏优先级：绝不隐藏名称；先隐藏类型，再隐藏大小；修改时间最后隐藏。隐藏列可从表头菜单恢复。

## 8.3 升级准备 / SQL

- 页面标题右侧 Primary：“生成升级资料”；未选择升级日期时禁用并给字段级说明。
- 一级 Tab：升级准备、SQL 整理、系统配置。Tab 位于标题下，独立 44px，不塞入卡片。
- 升级准备内容采用三段：顶部 64px 升级上下文（日期、系统、类型）；中部左 38% 候选需求，右 62% 发版任务编辑；底部 180–260px SQL/Excel 生成预览。
- 候选需求支持勾选、搜索、按计划上线推荐；系统切换时更新候选，不丢其它系统草稿。
- 生成动作始终固定在标题右侧或底部粘性操作栏右侧，不随滚动丢失。
- 960px 紧凑模式将左右分栏改为上“候选需求”下“任务编辑”，最小各 220px，可垂直拖动。

## 8.4 接口文档更新

- 标题右侧 Primary：“生成更新文档”。
- 顶部参数条按顺序：源文档路径（自适应）→ 输出目录（自适应）→ 日期 140px。
- 主体左右分栏：左 32% 输出文件浏览，右 68% 工作区。
- 右工作区上下分栏：上 60% SQL 输入/载入，下 40% 解析日志/预览。
- 小于 1120px：路径参数改为两行卡片，但主操作保持标题右侧；左右主体转 38/62。
- 输出文件表列“名称、修改时间、大小”，窄屏隐藏大小。

## 8.5 加解密 / XML / JSON

- 标题右侧显示“本地离线 · 内容不落盘”状态，不使用大块说明。
- 一级 Tab：国密解密、XML 格式化、JSON 查看；每个 Tab 有图标。
- 国密解密：参数区收为 52px 可折叠横条；输入/输出 50/50 可拖动，中央不放宽按钮列。主操作“开始解密”放输入卡片右上。
- XML：顶部 40px 工具栏，左侧粘贴/清空，右侧换行/复制；Primary“格式化 XML”固定在右。下方输入/输出 50/50。
- JSON：工具栏搜索自适应，路径复制靠右；树占主区域，原始文本作为可切换次级 Tab。

## 9. 按钮和操作位置统一规则

1. 页面主动作：标题区最右，整页只有一个实色 Primary。
2. 区域动作：区域标题右侧，使用 Secondary/Ghost，不抢页面主动作。

## 11. 响应式实现技术方案

PyQt 不使用网页 CSS media query，需要建立统一的窗口尺寸状态，而不是各 Panel 自行猜宽度。

### 11.1 建议新增

- `ui/layout_metrics.py`：所有宽度、间距、断点和控件高度常量。
- `ui/responsive.py`：根据主窗口可用宽度返回 `wide / standard / compact / narrow`。
- `MainWindow.layout_mode_changed` 信号：模式变化时通知各 Panel。
- Panel 可实现 `apply_layout_mode(mode)`，只做显隐、方向、尺寸策略切换，不重新创建业务控件。

### 11.2 断点

```python
WIDE = width >= 1440
STANDARD = 1280 <= width < 1440
COMPACT = 1080 <= width < 1280
NARROW = width < 1080
LOW_HEIGHT = height < 720
```

使用 100–150ms 单次防抖更新，避免拖动窗口时反复重排。模式未变化时不重复执行结构切换。

### 11.3 隐藏优先级

窗口缩小时按以下顺序减负：

1. 隐藏版本、副标题、快捷键等装饰/辅助信息。
2. 将低频筛选移入“更多筛选”。
3. 将纯说明文字收进 tooltip/帮助。
4. 将导航文字折叠，只保留图标。
5. 将左右双栏切换为上下双栏。
6. 最后才隐藏非关键表格列，并提供恢复入口。

绝不优先隐藏：当前页面标题、主操作、搜索、对象名称、关键状态、取消/确认按钮、错误结果。

### 11.4 状态持久化

只持久化用户主动调整的 splitter、列宽和导航折叠偏好；窗口因断点自动切换产生的临时宽度不覆盖用户的宽屏偏好。恢复时按当前模式裁剪到合法范围。

## 12. 微交互和科技元素

- hover 过渡 120–160ms；PyQt 无法直接实现时以颜色立即切换为主，不追求复杂动画。
- 激活导航使用轻柔 2px 顶部/左侧光带，不使用呼吸霓虹。
- 运行中状态使用 6px 青色状态点 + 简短文字；成功绿色、失败红色。
- Loading 浮层宽 320–520px，高 56px，位于内容区顶部居中 20px，不遮挡标题主操作。
- 表格选中态为浅靛蓝，当前焦点边界清晰；hover 与 selected 不使用相同颜色。
- 空状态采用 48px 线性图标、16px 标题、一句说明和一个明确动作，不展示大片空白。
- Toast 放右上内容区内，宽 320px，距顶部标题下方 12px，3–5秒消失；风险和需要决策的信息仍用 Dialog。

## 15. 验收画布与通过标准

必须分别在以下尺寸截图/人工查看：

- 1600×1000：宽屏信息密度、三栏/双栏使用是否舒适。
- 1440×900：设计基准，所有结构必须最完整。
- 1280×720：标准办公笔记本，文字导航仍清楚。
- 1100×700：图标导航/紧凑筛选触发正确。
- 960×640：最低尺寸，核心操作可达、无按钮重叠、无不可读压缩。

重点验收：

- 左侧为浅色导航，文字与图标清楚，不再压过内容区。
- 所有页面标题、筛选、主体的起始线一致。
- 每页只有一个明显 Primary，按钮按任务语义而不是代码顺序排列。
- 窗口缩放按明确断点变形，没有随机挤压或按钮文字消失。
- 需求管理右侧完成 Tabs 重排，不再纵向堆叠所有模块。
- 侧栏全部模块有一致图标，激活/普通/禁用颜色可读。
- Loading、弹窗、Toast、空状态不会推动布局。
- 现有业务信号、data 数据、SVN 边界、Private 打包规则保持不变。

任何只换侧栏背景色、只修改 QSS、只为部分模块补图标的交付均判定为不通过。本轮要求是应用壳层、响应式系统和核心页面结构一起落地。
