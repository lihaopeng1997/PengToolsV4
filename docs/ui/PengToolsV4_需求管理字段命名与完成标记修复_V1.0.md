# PengToolsV4 需求管理字段命名与完成标记修复 V1.0

## 目标

修复需求管理右侧详情中标记按钮被裁切的问题，并清楚区分“本事项是否涉及上线动作”与“该动作是否已经完成”。只调整 UI 呈现和可见文案；保持 `requirements.json` 的既有字段与联动逻辑不变。

## 一、右侧完成标记不可裁切

### 现状与根因

`RequirementPanel` 的 `flag_chips_layout` 是单行 `QHBoxLayout`。SQL、周边、接口、临时四个 chip 加 stretch 被强制放在同一行；右侧详情被用户拖窄或窗口处于最小宽度时，最后按钮只显示一半。

### 实现要求

- 用两列自适应网格替换单行 `QHBoxLayout`：最多两项一行，三/四项自然换到第二行。
- 每个完成按钮最小高度 `30px`，最小宽度约 `112px`；按钮文本不允许被省略或裁切。
- 右侧详情宽度低于 `520px` 时切换为单列；宽度恢复后回到双列。可使用 resize 重新排布，不要用固定窗口宽度硬截断。
- 没有适用事项时，整个完成标记区域隐藏；不显示空白容器。
- 为 960×640、右侧详情最窄允许宽度、四项均存在三种场景增加定向 UI 测试，断言四个按钮可见、几何区域完整且不重叠。

## 二、术语统一：上线事项 ≠ 是否完成

### 1. 编辑弹窗：选择适用的上线事项

原“升级标记（勾选后可在右侧点击切换完成状态）”含义混杂，改为：

- 分区标题：`上线事项`
- 辅助说明：`勾选本需求/BUG需要处理的事项；保存后可在详情中标记是否完成。`
- 四项字段：
  - `涉及 SQL`
  - `通知周边系统`
  - `更新接口文档`
  - `临时/紧急升级`

这些复选框仅表示“是否适用”，不表示完成与否。

### 2. 右侧详情：是否完成标记

- 在 chips 上方展示 11px 眉标：`完成标记`。
- 每个按钮完整表达状态，不再依赖只有红绿点的短文字：
  - `SQL · 待完成` / `SQL · 已完成`
  - `周边通知 · 待完成` / `周边通知 · 已完成`
  - `接口文档 · 待完成` / `接口文档 · 已完成`
  - `临时升级 · 待完成` / `临时升级 · 已完成`
- 点击只切换对应事项的 `flag_done[key]`，tooltip 明确说明“点击切换完成状态”。
- 为色弱可用性，状态不能只依赖红/绿：必须有文字和图标（待完成：时钟/圆点；已完成：勾选）。

### 3. 左侧树和右键菜单

- `无升级标记` → `无上线事项`。
- 右键菜单 `任务红绿点` → `完成标记`。
- `全部标完成` → `全部标为已完成`；`全部标待办` → `全部标为待完成`。
- 左侧 tooltip 使用 `待完成 / 已完成`，不要写“红点/绿点”。

## 三、需求详情字段名审计与调整

| 当前展示 | 调整后 | 原因 |
|---|---|---|
| 类型 | 事项类型 | 明确包含需求和 BUG |
| 当前状态 | 进度状态 | “当前”没有信息增量，状态本质是进度 |
| 所属系统 | 目标系统 | 更符合本次需求/BUG归属与升级准备选择 |
| 上线时间 | 动态字段名：实际上线 / 计划上线 / 上线月份 | 当前值按实际、计划、月份回退，但标题固定“上线时间”会误导 |
| 已绑定资料 / 未绑定资料 | 资料已绑定 / 未绑定资料 | 状态表达更直接 |
| 文件与 SVN | 文件与版本库 | 让不熟悉 SVN 的用户也理解；完整 SVN 路径仍保留 |
| 关联 SQL | SQL 脚本 | 这是实际内容而非抽象关系 |
| 上线与联动 | 发布与联动 | 覆盖上线、日报、接口文档、升级准备等动作 |
| 整理 SQL | 打开 SQL 整理 | 明确是跳转，不是此处直接修改 |
| 发送接口文档 | 更新接口文档 | 与上线事项字段一致 |
| 进入升级准备 | 准备本次升级 | 更贴近操作结果 |

### 动态上线字段规则

- `actual_online_date` 非空：显示字段名 `实际上线`，显示实际日期。
- 否则 `planned_online_date` 非空：显示字段名 `计划上线`，显示计划日期。
- 否则 `online_month` 非空：显示字段名 `上线月份`，显示 `YYYY年M月`。
- 都为空：显示字段名 `计划上线`，值为 `未安排`。
