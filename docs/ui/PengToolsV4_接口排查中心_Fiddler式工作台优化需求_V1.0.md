# PengToolsV4 接口排查中心：Fiddler 式工作台优化需求 V1.0

## 1. 目标与边界

将已完成的「多浏览器接口排查中心」升级为适合日常问题定位的 Fiddler 式三栏工作台：快速发现请求、快速筛选失败接口、快速阅读报文、快速形成 Postman/cURL 本地验证草稿。

本次为体验和信息组织升级，不改变既有安全边界：

- 仅 Private 版；标准安装包不动。
- **不内置、不打包任何浏览器 EXE 或浏览器目录**，避免安装包膨胀、授权与浏览器升级维护问题。
- 保留「自动识别本机浏览器」「手动选择浏览器 EXE」「连接已有调试端口」；仅启动用户本机已有的浏览器。
- Chromium 仅连接 `127.0.0.1` 调试端口；IE 仅使用用户明确确认后的本机代理模式。
- 不发送任何业务请求；本地验证只生成 URL 重写、Postman Collection 与 cURL 草稿。
- 捕获报文、Cookie、Token、密钥只存内存，停止监听或退出软件必须清空；仅允许保存浏览器路径、端口、过滤偏好、本地地址配置和证书指纹。

## 2. 整体布局

基于现有 `panels/interface_debug_panel.py` 改造，不新建第二套业务逻辑。

```text
┌────────────────────────────────────────────────────────────────────────────┐
│ 接口排查  本机监听 · 仅内存 · 草稿验证                         [状态胶囊] │
├────────────────────────────────────────────────────────────────────────────┤
│ 浏览器 / 连接控制区：模式 | 浏览器 | 端口 | 启动 | 开始监听 | 停止 | 更多 │
├────────────────────────────────────────────────────────────────────────────┤
│ 会话工具条：搜索 | 全部 | XHR/Fetch | 失败 | JSON/XML | 静态资源 | 清空    │
├───────────────────────┬────────────────────────────────────────────────────┤
│ 请求会话列表          │ 请求详情 Tab：概览 | 请求 | 响应 | 验证草稿         │
│ 时间 方法 状态 路径   │                                                        │
│ 域名 耗时 类型 大小   │ 结构化内容 / 代码文本 / 操作按钮                      │
└───────────────────────┴────────────────────────────────────────────────────┘
```

- 默认左右比例 42% / 58%，可拖动并按当前响应式模式保存。
- 不采用 Fiddler2 的密集灰色表格复刻；保持 PengTools 的企业级、安静、可读风格。
- 所有高频动作在可见区域完成；低频配置、证书、导出和列设置收进「更多」。

## 3. 会话列表（左侧）

### 3.1 列与排序

替换现有六列为下列可排序列，默认按最新请求在顶部：

| 列 | 默认可见 | 说明 |
|---|---:|---|
| 状态 | 是 | 以色点 + HTTP 状态码表示；失败使用危险色，不只靠红字 |
| 方法 | 是 | GET/POST/PUT/DELETE 等固定宽度 badge |
| 接口路径 | 是 | 展示 `host + path`，超长省略且 Tooltip 显示完整 URL |
| 耗时 | 是 | 毫秒；超过 1s 使用警告色，超过 3s 使用危险色 |
| 类型 | 是 | JSON/XML/HTML/图片/脚本/其他 |
| 时间 | 是 | `HH:mm:ss.SSS` |
| 大小 | 否 | 响应体可得时展示格式化大小 |
| 来源 | 否 | Chromium / IE |

- 用户可在「列设置」中恢复或隐藏次要列；列宽与排序选择本地保存。
- 表格行高 32px，状态列 72px，方法列 66px，耗时列 74px；路径列优先伸展。
- 无请求时展示空状态：引导先选择浏览器、启动或连接调试端口、然后正常操作网页。

### 3.2 筛选与搜索

- 搜索实时过滤：匹配 URL、host、path、method、状态码、Content-Type；输入 150ms 防抖。
- 增加可组合 FilterChip：`全部`、`XHR/Fetch`、`失败(4xx/5xx)`、`慢请求(>1s)`、`JSON/XML`、`静态资源`。
- 默认隐藏静态资源；用户勾选后展示。
- 显示会话计数，例如「38 / 126 条」；清空操作需确认，取消在左、确认清空在右、默认焦点为取消。
- 选中后保持列表位置，不因新报文到来跳走；仅在用户停留于首行时允许自动跟随最新请求。

## 4. 详情区（右侧）

将当前「请求 / 响应」两个 Tab 扩展为四个 Tab：

### 4.1 概览

- URL、方法、状态、耗时、类型、来源、开始时间、响应大小。
- URL 展示默认脱敏；显示「复制安全 URL」和「显示敏感内容」开关。显示敏感内容必须沿用现有确认机制，且本会话有效。
- 请求状态、慢请求、空响应、响应体不可获取等状态使用清晰解释，不把 CDP 限制伪装成异常。

### 4.2 请求

- 分区呈现：Query 参数、Headers、Cookie、Body。
- Headers/Cookie 默认脱敏；每个分区提供复制按钮，复制敏感信息前弹出一次提示。
- JSON/XML Body 自动识别，显示「格式化查看」「送入加解密」「复制」；普通文本原样显示。

### 4.3 响应

- 分区呈现：响应头、Body。
- JSON/XML 自动格式化预览；格式错误时保留原文并展示原因。
- 显示「送格式工具」「送入加解密」「复制响应」；不得自动解密，也不得写入磁盘。

### 4.4 验证草稿

- 沿用现有本地地址配置、地址重写、Postman v2.1 和 cURL 生成功能。
- 将本地地址、重写后的 URL、携带 Cookie/认证头开关、输出格式集中展示。
- 默认完整生成认证头/Cookie，但 UI 保持脱敏；复制或导出前明确提示敏感信息。
- 主按钮为「生成草稿」，次级按钮为「复制 Postman」「复制 cURL」「导出 Collection」；明确标注「不会发送请求」。

## 5. 浏览器与连接控制

### 5.1 Chromium

- 浏览器选择下拉显示：图标、名称、路径短名；完整路径在 Tooltip。
- 自动扫描常见安装位置；保留「刷新识别」与「选择浏览器 EXE」。
- 手选路径保存为最近使用项，可删除；不复制、不打包该 EXE。
- 「启动调试浏览器」使用独立资料目录与本机端口，不影响日常浏览器资料。
- 「连接已有浏览器」仅接受 `127.0.0.1` 端口；端口不可用时给出下一步操作提示。
- Firefox 保持明确提示：首版仅支持 Chromium 实时监听，不假装兼容。

### 5.2 IE

- 保持现有高风险二次确认、WinINet 备份/恢复、用户证书安装/移除、异常退出恢复提示。
- IE 控件仅在 IE 模式显示，避免 Chromium 模式顶部出现无关按钮。
- 启动监听后，状态条固定显示「代理仅作用于当前用户；停止后自动恢复」。

### 5.3 长任务与响应式

- 启动浏览器、连接 CDP、启动/停止 IE 代理、安装/移除证书使用独立 Loading 浮层；不得推动顶部按钮位置。
- Wide/Standard：完整连接控制区。
- Compact：显示模式、浏览器、启动、监听、停止、更多；端口和证书管理进入更多或第二行。
- Narrow：顶部仅显示连接状态、开始/停止、更多；浏览器选择与端口进入抽屉/更多菜单。
- 详情区在 Compact/Narrow 改为上下 splitter，保证报文编辑区最小高度 180px。

## 6. 主题、图标与状态

- 完全使用 PengTools ThemeManager token，兼容四套主题，尤其是夜间安读；禁止硬编码白色背景。
- 图标复用现有 `resources/icons/api-debug.svg` 体系；状态使用简洁圆点、方法 badge 和语义色，避免堆砌装饰图标。
- Loading、弹窗、右键菜单与夜间主题一致性优化需求同步遵守。

## 7. 技术实现要求

- 捕获、合并、脱敏、草稿生成继续放在 `tools/`；Panel 只负责展示与交互。
- 复用现有 `BrowserDebugSession`、`IeProxyWorker`、`interface_drafts.py`，禁止重复创建网络发送能力。
- 新增会话筛选/排序/视图状态应可定向单测，不保存报文内容。
- 结合 `ui/responsive.py` 的 layout mode；实现 `InterfaceDebugPanel.apply_layout_mode(mode, low_height)`，实际重排，不只记录模式变量。
- 本地配置新增字段必须 `setdefault` 兼容旧版 `data/interface_debug.json`；仅保存非敏感 UI 偏好。
