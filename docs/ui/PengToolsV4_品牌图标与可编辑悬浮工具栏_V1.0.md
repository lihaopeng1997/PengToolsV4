# PengToolsV4 品牌图标与可编辑悬浮工具栏

> 本文为下一轮开发硬性需求。  
> 关联资源：`resources/brand/`。  
> 目标：将悬浮窗从固定功能清单，改为可由用户配置的个人快捷入口；同时统一应用、任务栏、托盘和悬浮图标。

## 1. 品牌标识

使用 `resources/brand/README.md` 中定义的“收拢的工作流”标识。它由起点圆点、回折路径和落地短线组成：既可被识别为 P，也表达需求、资料、SQL、发版最终收拢到一个工作台。

### 资源对应

| 场景 | 使用资源 | 规则 |
|---|---|---|
| 应用窗口 / Windows 任务栏 / 安装程序 | `resources/brand/pengtools-app-v2.ico` | 替换旧 `resources/app.ico` 的引用；使用多尺寸 ICO，不再显示旧图标 |
| 应用 Logo / 侧栏品牌区 | `resources/brand/pengtools-app-mark.svg` | 36px 品牌底板内显示 24px 标识 |
| 系统托盘 | `resources/brand/pengtools-tray-mark.svg` | 16/20px 单色标识，按当前主题染色；删除现有圆形 P 回退绘制 |
| 悬浮窗收起按钮 | `resources/brand/pengtools-floating-mark.svg` | 32px 标识置于 44px 点击区；删除文字 `P` 与字符 `×` |

应用主色由当前主题决定，但 Logo 的结构不可因主题而变化。默认“静谧办公”下任务栏图标使用鼠尾草绿底 + 暖白路径；夜间主题仅染托盘/悬浮单色标识，不替换 Windows 已固定的 ICO。

## 2. 悬浮工具栏的产品定位

悬浮工具栏不是第二套导航，也不是把所有功能逐条罗列。它是“此刻最常用的个人快捷入口”。

规则：

1. 收起态只显示 Logo 标识，保持 52×52px，拖动不变。
2. 展开态只显示用户启用的快捷入口，最多 6 项；不显示完整模块目录。
3. 默认快捷入口：需求管理、升级准备、日报、加解密。
4. 首页固定为展开面板底部的弱操作“打开完整工作台”，不占一个用户快捷位。
5. 自我学习仅在彩蛋已解锁后才可进入可选清单；日报和需求管理始终允许选择。
6. 快捷入口改变的是打开目标，不改变模块可见性、业务数据、权限或彩蛋规则。

## 3. 收起态与展开态布局

### 收起态

- 总尺寸：52×52px；可点击核心区域：44×44px；外边距 4px。
- 背景：当前主题 `surface`，细边框，轻阴影；不使用大圆形高饱和色。
- 中间为 `pengtools-floating-mark.svg`，尺寸 28–30px。
- hover 时仅提升边框/阴影，不做旋转、闪烁或强动效。
- tooltip：`打开快捷工具`；拖动时不触发打开。

### 展开态

- 标准宽度：300px；默认高度 250–350px，根据 1–6 个入口自适应；不再固定 512px。
- 面板向左展开并保持收起 Logo 的屏幕位置不跳动。
- 顶部 48px：左为 Logo + `快捷工具`，右为铅笔图标“编辑快捷入口”和收起图标。
- 中间：两列快捷卡片网格；每卡高度 58px，图标 20px，模块名 13px；四项时为 2×2，六项为 2×3。
- 卡片只显示图标和模块名，禁止显示复杂副标题；hover 显示当前模块一句 tooltip。
- 底部 40px：左侧 `打开完整工作台`，右侧可选 `设置快捷入口`，使用幽灵按钮。
- 宽度不足靠近屏幕左侧时向右展开；靠近底部时向上展开；不得越出当前屏幕可用区域。
- 展开状态点击面板外、按 Esc 或点击收起图标即可收起。

## 4. 编辑快捷入口

编辑入口提供两种等价方式：展开面板右上角铅笔，或设置 → 悬浮工具栏 → 编辑快捷入口。二者修改同一份本地配置。

### 编辑弹层

- 宽度 520px，最大高为屏幕 70%，标题为“编辑快捷入口”。
- 顶部说明：`选择常用功能，最多显示 6 个。拖动可调整展开后的顺序。`
- 中部为可拖动列表：每行 48px，左侧拖动柄，中间模块图标+名称，右侧开关/勾选框。
- 列表包含：首页（只可作为底部入口，不占快捷位）、需求管理、升级准备、接口文档更新、日报、加解密、证件类型、车辆 VIN、运维助手，以及解锁后可见的自我学习。
- 至少保留 1 个快捷入口；当达到 6 项时，其余未选项禁用并提示“最多 6 项”。
- 变更即时保存；底部只保留“恢复推荐”弱按钮和“完成”主按钮。取消只取消尚未保存的编辑会话。

### 懒人体验

- 首次使用无需配置，默认四项即可开始。
- 用户不必理解编号或模块 index；使用自然语言名称和图标。
- 用户取消勾选不会删除模块，只从悬浮快捷区移除。
- 恢复推荐始终可恢复为：需求管理、升级准备、日报、加解密。

## 5. 数据和技术实现

1. 在 `config.DEFAULT_SETTINGS` 新增 `floating_shortcuts: [10, 2, 9, 5]`，按导航 index 存储；`normalize_settings()` 去重、过滤非法 index、过滤未解锁的自我学习，并保证至少一个有效入口。
2. 建立共享的模块元数据（推荐 `ui/navigation_model.py`）：`index、中文名、英文名、图标角色、是否可悬浮、是否需彩蛋`。`MainWindow`、`QuickPanel`、设置编辑器必须从同一来源读取；禁止重复维护三套模块名称和图标映射。
3. `QuickPanel` 不再使用 `range(1, 8)` 固定生成按钮；按 `floating_shortcuts` 动态创建卡片，刷新配置后重建布局但保留面板位置和展开/收起状态。
4. 编辑过程中只修改设置副本；点击完成后调用既有 `save_settings()`，由 `MainWindow` 将最新配置应用到 `QuickPanel`。
5. 用 `QIcon`/本地 SVG 加载模块图标；收起开关用 `pengtools-floating-mark.svg`，不得依赖文本 P。
6. `TrayService` 直接加载品牌托盘 SVG 的主题 tint pixmap；若 SVG 失败，回退 `pengtools-app-v2.ico`，不再绘制旧的蓝色圆形 P。
7. `run.py`、`PengToolsHub_Private.spec` 和 `build_private_release.ps1` 全部切换到新的 ICO、PNG、品牌 SVG 目录；Private 包必须包含 `resources/brand`。
8. 不修改标准安装包，不将用户 `data/` 打进包，不引入在线资源。

## 6. 定向验收

- Windows 任务栏、窗口标题栏、Private 安装包均显示新 Logo；16px 下路径和起点圆点仍清楚。
- 托盘显示单色新标识，切换主题后颜色正确更新；右键菜单和双击打开主窗口保持可用。
- 悬浮收起态不再显示字母 P；展开态高度随 1–6 个入口变化，不出现空白长面板。
- 默认显示需求管理、升级准备、日报、加解密；点击均能进入正确模块。
- 编辑中可勾选、取消、拖动排序、恢复推荐；重启软件后仍保留。
- 未解锁时自我学习不出现在编辑清单；解锁后才可添加。
- 任务栏、托盘、悬浮窗、侧栏品牌图标视觉同源；无旧蓝色圆 P、无 Emoji、无文字替代 Logo。
