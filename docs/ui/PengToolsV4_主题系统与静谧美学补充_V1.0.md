# PengToolsV4 主题系统与静谧美学补充

> 本文补充并覆盖 `PengToolsV4_企业级界面全量布局蓝图_V2.0.md` 中的“科技感”取向。  
> 产品方向：安静、愉悦、精致、长期使用不疲劳的个人工作台。  
> 主题入口：设置 → 外观 → 主题。

## 1. 主题系统目标

主题不是简单的背景换色，也不是把软件做成炫目的皮肤软件。每套主题必须完整覆盖应用背景、侧栏、卡片、文字、边框、按钮、下拉框、表格/Tree、状态色、弹窗、Loading 和 SVG 图标染色；主题之间不改变布局、功能位置、操作路径和数据。

默认使用“静谧办公”。用户切换后即时预览、即时生效、自动保存到本地设置；无需重启，不影响 `data/`、分栏尺寸、列宽、SVN 或业务数据。

首版只提供经过可读性检查的四套内置主题，不开放任意调色和自定义 QSS，不从网络下载主题。

## 2. 四套主题方向

### 2.1 静谧办公（默认）

这是默认主题，也是全软件的审美基准。

- 气质：像整理得很好的安静办公室，留白充足、稳定、克制。
- 背景：暖灰白 `#F6F5F1`。
- 侧栏：米白 `#FBFAF7`，不再使用深色。
- 卡片：纯白 `#FFFFFF`，细暖灰边框 `#E2E0D9`。
- 主色：鼠尾草绿 `#668C78`；选中弱背景 `#E9F1EB`。
- 标题：深石墨 `#272B29`；正文 `#424A45`；辅助文字 `#7B847E`。
- 适合：需求管理、升级准备、日报，是用户每天打开软件时的默认心情。

### 2.2 晴空清晰

- 气质：清爽、干净、理性，适合信息密度高的工作。
- 背景：雾白蓝 `#F4F7FB`；侧栏 `#FAFCFF`。
- 主色：低饱和灰蓝 `#547A9D`；选中弱背景 `#E8F0F7`。
- 适合：需求目录、SQL、发版 Excel、文件表格。

### 2.3 暖书房

- 气质：温暖、有纸张和阅读感，不甜腻、不复古过头。
- 背景：浅燕麦 `#FAF7F2`；侧栏 `#FFFDF9`。
- 主色：焦糖棕 `#A87950`；选中弱背景 `#F5EBDD`。
- 适合：自我学习、日报、接口文档、长文本阅读。

### 2.4 夜间安读

- 气质：安静的深色阅读，而不是赛博终端。
- 背景：墨绿灰 `#202523`；侧栏 `#29302C`；卡片 `#2E3732`。
- 主色：柔雾绿 `#9ABAA6`；正文 `#E7ECE7`；辅助文字 `#B8C1B9`。
- 高风险色、成功色、表格边界都降低饱和度，避免深色环境刺眼。
- 适合：夜间看 SQL、XML、日志、Word 长文档。

## 3. 设置页设计

“设置 → 外观”顶部固定展示“主题”区域，放在字体大小之前。

```text
主题
选择一个更适合当前工作状态的界面外观。布局和数据不会改变。

[ 静谧办公 ] [ 晴空清晰 ] [ 暖书房 ] [ 夜间安读 ]
  当前使用
```

每张主题卡为 104 × 72px：上部 44px 展示真实背景、卡片和主按钮色块；下部展示主题名。当前卡片使用 2px 主色边框、对勾和“当前使用”文字；非当前卡片 hover 时提升边框对比，不使用跳动动画。

在主题卡下方显示一行说明：`主题仅改变外观，不会影响文件、数据、SVN 与功能位置。`

字体大小、界面语言继续位于同一“外观”分类，但不再与主题卡混成普通表单行。窗口窄于 1080px 时，主题卡可以 2×2 排列；不允许压缩成一排过小色块。

## 4. 按钮与模块展示的美学规则

1. 每个页面只保留一个实色 Primary 按钮；其余为描边、浅底或图标按钮。
2. Primary 默认使用主题主色，圆角 8px，高 36px，不做高光、渐变、粗阴影。
3. 当前导航采用主题弱背景 + 主色文字/图标；不用大面积深色底、霓虹色条或发光。
4. 卡片使用白色/主题表面、1px 边框、12px 圆角；只在弹窗、菜单、浮层使用轻阴影。
5. 表格和目录以留白、行高、浅色选中态建立秩序；不对每一行使用强色块。
6. 成功、警告、危险只表达状态，不承担装饰：成功柔绿、警告低饱和琥珀、危险砖红。
7. 空状态只展示一枚线性图标、一句说明和一个行动按钮；不要插入夸张插画或营销文案。
8. 禁止霓虹、扫描线、粒子、闪烁、玻璃拟态、大面积渐变、赛博发光和 Emoji 图标。

## 5. 实现边界

1. 新增 `ui/theme_manager.py`，由 `ThemePalette` 管理四套 token；它是唯一主题入口。
2. `resources/style.qss` 改为 token 模板，例如 `__APP_BG__`、`__SURFACE__`、`__PRIMARY__`、`__TEXT__`，禁止 Panel 写主题色。
3. `run.load_stylesheet()` 只读取样式模板；`ThemeManager.apply(app, theme_id, font_size)` 注入主题、图标资源路径和字体大小，避免反复拼接 QSS。
4. `config.DEFAULT_SETTINGS` 增加 `ui_theme: "calm"`；`normalize_settings()` 对旧配置补默认值并保留未知字段。
5. `ui/icons.py` 的缓存 key 必须包含主题 ID 和 tint；主题变更时刷新导航、按钮、Dialog 徽章等动态图标。
6. `MainWindow` 负责一次性广播主题变更给 Panel；Panel 只刷新自绘内容或动态生成图标，不保存自己的主题状态。
7. 主题切换失败需回退到上一个有效主题并显示轻量提示；不丢失用户设置。

## 6. 验收

- 初次启动默认“静谧办公”，旧数据升级后也正确显示。
- 四个主题都能即时生效，重启后保留。
- 所有页面、弹窗、菜单、Loading、Tree、Table、Tab、下拉框和图标均随主题变化，无残留旧色。
- 日间主题下侧栏文字清楚；夜间主题下表格、输入框、危险按钮仍可读。
- 主题切换不改变页面布局、导航顺序、当前模块、数据和业务功能。
- 960 × 640 时主题卡仍可辨识、可点击且无重叠。
