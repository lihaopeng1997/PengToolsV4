1. 产品简介与安装

PengToolsHub 是面向内网办公的 离线 Windows 工具集合，把日常开发、运维、需求归档、SQL 发版、接口排查、加解密等流程收敛到一个本地窗口里。

1.1 能做什么

  • 管理需求 / BUG、SVN 工作副本与文件库
  • 整理 SQL、生成发版 Excel
  • 写日报、（彩蛋解锁后）管理自我学习资料
  • 网关 SM2+SM4 加解密、JSON/XML 格式化
  • 本机 HTTP/HTTPS 抓包、按环境做请求测试

1.2 安装与启动

  1. 解压私人安装包 PengToolsHub_Private_Offline_Setup.zip
  2. 运行目录中的 PengToolsHub_Private.exe（或 setup.cmd 按说明）
  3. 首次运行会在 EXE 同级 自动创建 data/ 目录存放配置与业务数据
提示：程序支持单实例。重复启动会激活已打开的窗口，不会开第二套托盘/后台服务。

3. 首页工作台

  • 最近需求：按更新时间展示最近几条，点击跳转需求管理并定位。
  • 待升级事项：只列出已填写上线日期且日期 ≥ 今天、尚未实际上线、状态非「已上线/关闭/取消/暂停」的需求。
  • 今天上线的需求会高亮显示，并优先排在前面。
  • 常用工具：一键跳转加解密、证件、文档、VIN、运维等。

4. 需求管理

4.1 左侧需求树

  • 按月份/类型组织需求与 BUG
  • 支持搜索（含拼音）、筛选系统/类型/状态
  • 批量选择与删除（删除有二次确认，默认焦点在「取消」）

4.2 右侧摘要与页签

  • 上方摘要：标题、系统、状态、计划上线日、标志芯片等
  • 文件库：名称 / 类型 / 时间 / 大小 / 路径；仅名称列带图标；可拖列宽、横向滚动、拖入本地文件到工作副本
  • SQL / 发版联动入口、发布与联动（写入日报、准备升级、接口文档等）

4.3 SVN 相关

  • 检出、更新、提交、锁定/解锁等依赖本机已安装的 SVN 命令行
  • 认证使用本机缓存，软件不保存密码
  • 内网 SVN 需在你本机环境验证
注意：破坏性操作（删除、提交等）均有确认框；请确认后再执行。

5. 发版联动（升级准备）

  1. 确认/选择升级日期
  2. 推荐或勾选参与发版的需求 / BUG
  3. 按多系统分别整理 SQL
  4. 生成发版 Excel（基于内置模板，23 列表头固定）

开发分支 SVN 地址可以为空，不会阻塞生成。验证用 SQL 不会进入 SVN 提交目录逻辑。

6. 日报

  • 左侧历史日期列表，右侧编辑：今日完成 / 问题与风险 / 明日计划 / 备注
  • 未点保存就切换日期：内容会作为草稿保留，列表可能显示「未保存」
  • 复制为今日：把当前正在看的内容一键写成「今天」的草稿，确认后请点「保存日报」
  • 从需求「写入日报」会追加模板段落，不覆盖你已写内容
  • 提醒时间在「设置」中配置

7. 网关加解密

  • 用于网关 SM2 + SM4 类报文解密与查看
  • 从接口排查「送入加解密」时可自动带入报文与 Key（能识别时）
  • 密钥 / 明文 / 报文默认不落盘、不写日志

8. 接口排查（Private）

8.1 开始抓包会做什么

  1. 在本机 127.0.0.1:端口 启动 MITM 代理
  2. 临时修改 Windows 系统代理指向该地址（便于 Chrome/Edge 等走代理）
  3. 状态栏提示抓包中；请用浏览器访问业务页（必要时完全退出并重启浏览器）
重要：抓包会改系统代理。请务必用「停止抓包」或正常退出软件，代理会自动恢复。 若任务管理器强杀进程，可能导致代理残留，其它接口会「不好使」。

8.2 停止 / 恢复代理

  • 停止抓包：恢复系统代理，会话列表默认保留（不清空）
  • 清空：才清空内存中的抓包记录
  • 恢复系统代理按钮：异常后一键修复
  • 软件启动时也会自动检测并清理残留的本机抓包代理

8.3 列表与详情

  • 列表：# / 结果 / 协议 / 方法 / 主机 / URL / Body / 类型 / 耗时 等（可列设置）
  • 详情：概览 / 请求 / 响应 / 请求测试
  • 可复制、送格式工具、送入加解密（尽量带 Key）

8.4 请求测试（Postman 风格）

  1. 维护环境：Base 形如 http://host:port（可保存多个）
  2. 从会话填充：自动用当前环境替换抓包 URL 的 host，保留 path/query
  3. 编辑 Method / Headers / Params / Body 后点「发送」
  4. Body 优先使用能解密的明文；解不了则用原报文

8.5 导出 / 导入明细

  • 导出格式：pengtools_iface_session_v1（JSON）
  • 内容含 URL、请求/响应（优先解密明文）等
  • 可再「导入明细」或把 JSON 拖入请求测试页自动填充
安全：抓到的请求/响应/Cookie/Token 只在内存；配置里只存端口、环境地址、证书指纹等，不写报文正文。

12. 常见问题 FAQ

Q1：抓包后列表是空的？

  • 确认已点「开始抓包」且状态为抓包中
  • 完全退出浏览器再打开，再访问业务页
  • HTTPS 需信任本机抓包证书（软件会尽量静默准备）
  • 检查筛选条件是否过滤掉了静态资源/全部请求

Q2：抓包后网页/其它接口都不好使？

  • 点「停止抓包」或「恢复系统代理」
  • 或重启一次 PengToolsHub（启动会自动清理残留代理）
  • 仍不行：Windows 设置 → 网络和 Internet → 代理 → 关闭「使用代理服务器」

Q3：请求测试发不出去？

  • 检查环境 Base 是否为 http(s)://host:port 且不带业务 path
  • 目标服务是否可达（本机/内网）
  • HTTPS 内网自签证书：工具侧已尽量放宽校验，仍失败请看响应区错误信息

Q4：日报写了但刷新没了？

请点「保存日报」。未保存内容在切换日期时会暂存草稿，但退出进程后只有已保存的 JSON 会保留。

Q5：发版 Excel 列不对？

请使用软件内置模板生成，勿手工改 23 列表头顺序。
